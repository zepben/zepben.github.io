[1.0.0]ŌĆŗ

Breaking ChangesŌĆŗ

  • PRRP: Replaced field voltageDeltaAvgThreshold with averageVoltageSpreadThreshold in CandidateGenerationConfig.TapOptimization for clarity. This serves the same purpose, but is in volts instead of per-unit voltage.
  • The input schema must be version v20260223.
  • Minimum required JVM version for compilation is 17.
  • Executor: DVMS voltage inputs are now in per nominal LV voltage base rather than per unit, the config field names have not been updated to reflect this.
  • Executor: Nominal LV voltage base is now configured by the work package.
  • Removed all CLI options except for -c/--config.

New FeaturesŌĆŗ

  • Consolidated in opendss-jvm-executor. Future changes to the executor will happen in this repository.
  • Consolidated in opendss-results-processor. Future changes to the executor will happen in this repository.
  • Consolidated in prrp. Future changes to the executor will happen in this repository.
  • Consolidated in opendss-exporter-jvm. Future changes to the library will happen in this repository.
  • Added entrypoint for Standalone Executor.
  • Added entrypoint for Standalone Results processor.
  • Added entrypoint for Standalone Post Results Results Processor (Prrp).
  • PRRP: The nominal phase-to-phase voltage for LV customers can now be configured via nominalLowVoltage, defaulting to 400. This is used when calculating tap optimisations.
  • Added ability to mock feeder head voltage via new config field loadDb.mockedFeederVoltagePerUnit.
  • Executor: To assist with debugging DVMS can output the final voltage and customer count information of each step to a csv file. Enabled via the config DvmsConfig.outputToCsv.
  • Added negligibleImpedanceMinHvThreshold and negligibleImpedanceMinLvThreshold to ModelConfig to define negligible impedance during network simplification.

EnhancementsŌĆŗ

  • PRRP: Improved hard-coded per-nominal-LV voltage limits in tap optimisation algorithm.
  • Result Processor: Make nullable parquet config options default to null.
  • Enhancements to StorageControllers generated in BESS interventions:
    • The deadband for both targets are now set for peakshave
    • The deadbands are set as a percentage of the rated kVA of the target equipment (transformer/switch) rather than the target
    • The target power levels for charging/discharging are set as a percentage of the rated kVA of the target equipment (transformer/switch) rather than an absolute kW value
  • OpenDssDiskWriter now suffixes model paths by the current timestamp when no outputModelId is provided.
  • If no outputModelId is specified for an OpenDssDiskWriter, the timestamp when the model was generated is used instead.
  • Executor: Log the stacktrace of unhandled exceptions encountered during execution.
  • Executor: DVMS will now attempt to find the neutral tap position and use that as the initial setting, if not able, the tap position in the middle of the transformer's range will be used.
  • Executor: DVMS will allow 5x the maxDVMSIterations on the first time step to ensure that DVMS can reach the desired tap position from the initial setting.
  • Executor: Optimised retrieval of load voltages for DVMS.

FixesŌĆŗ

  • Updated code to catch previously uncaught errors in the process of generating a model.
  • CommonImpedanceCombiner and ImpedanceNormalizer will no longer stack overflow on large traces. (from commons v1.1.2)
  • PostgresLoadDataFetcher ignores virtual usage points when requesting load data for UsagePoints or EnergyConsumers. A LoadData with null load is returned if all usage points for the request are virtual. (from commons v1.1.2)
  • Lines generated above EnergyConsumers during network simplification are no longer missing from the OpenDSS model.
  • Fixed bug where collapsed objects would be assigned the same UID if they were collapsed from the same equipment. This resulted in only one of them being written in the OpenDSS model. This bug was fixed by prefixing the UID with the mRID of the collapsed object.
  • PRRP: Tap optimisation candidate generation now operates against per-nominal-LV voltages instead of per-unit, where nominal LV is configurable.
  • PRRP: Fixed crash that happens when load_exceeding_normal_thermal_max_kw or gen_exceeding_normal_thermal_max_kw is infinite
  • Executor: Fix the supervisor hanging when sending a failure message
  • Fixed convergence issues caused by incorrect phasing on batteries when using the COMMUNITY_BESS intervention.
  • useMRIDsAsNMIs option is now only used for PQV, since load data is now queried by usage point mRID.
  • Explicitly close customer consumer client during model generation.
  • Fix the results processor hanging when the executor failed due to a segmentation fault.

NotesŌĆŗ

  • Project upgraded to Kotlin 2.3.20

    • Profiles for CPIs are absolute, so a ratedPower of 100000 means no load scaling and 150000 means 150% load scaling.

  • The built jar has been changed to the unified runtime entrypoint. Commandline arguments are no longer accepted, And any existing config files will need to be updated. The entry point config for the existing functionality is StandaloneGenerator, any existing configuration can be passed to the config object of the entrypoint block.

New FeaturesŌĆŗ

  • Support for writing OpenDss model to blob storage.
  • Support for reporting progress including individual models processed to RabbitMq queue.

EnhancementsŌĆŗ

  • Retry RabbitMQ connections during the initial connection procedure.
  • Increased load scaling precision to ┬▒0.001% of the base load.
  • Enable RabbitMqClient connections to be retried automatically 3 times.

FixesŌĆŗ

  • When creating RegControls always use phase to ground voltage for r/x and revR/revX.
  • When collapsing conductors:
    • We now calculate whether a conductor is negligible impedance z, rather than independently checking r and x values.
    • We will now leave in place an AcLineSegment (with minimum impedance) above EnergyConsumers to avoid connecting EnergyConsumers directly to a branch in the network, which stops us grouping consumers under a single EnergyMeter in calibration models.
  • Log RabbitMQ connection failures as ERROR
  • Default loads are no longer created at aggregate load points (distribution transformers).

  • Updated to com.zepben.opendss-exporter v0.8.0

  • Updated to com.zepben.commons v1.3.0

  • Include OpenTelemetry dependency for RabbitMQ client metrics.

    • [0.8.0] Profiles for CPIs are absolute, so a ratedPower of 100000 means no load scaling and 150000 means 150% load scaling.

[0.7.2] - 2025-12-12ŌĆŗ

Breaking ChangesŌĆŗ

  • ratedPower of usage points are now interpreted to correspond to per-0.001% scaling applied to the profile backing each one, rather than percent scaling:
    • Loadshapes for DER are normalised in the input DB, so ratedPower is set to the peak power of the corresponding PowerElectronicsUnit in per-0.01VA, which is equivalent to per-0.001% scaling on top of a 1kVA-peak profile.
    • Profiles for CPIs are absolute, so a ratedPower of 100000 means no load scaling and 150000 means 150% load scaling.

[0.7.0] - 2025-08-02ŌĆŗ

Breaking ChangesŌĆŗ

  • Updated to use new zepben commons library com.zepben.commons. This should not be breaking on output, but is high risk so act like it is.
  • Removed factory function suspend (Feeder, Double) -> LoadScalingFactor? from type alias CimTranslatorCreator, since load scaling factors are now inferred from the CIM model after applying a scenario, via the ratedPower on UsagePoints. This affects createCimToOpenDss: CimTranslatorCreator in ExplicitProcessor and ExplicitStorageBackendProcessor.

New FeaturesŌĆŗ

  • Added support for node level results. Configured via adding a NodeLevelResults object to the nodeLevelResults field of GeneratorConfig.
  • Added support for controlling the simplification of the CIM model prior to OpenDSS model generation. The new Boolean fields (all true by default) in ModelConfig are:
    • simplifyNetwork - Setting to false completely disables network simplification.
    • collapseNegligibleImpedances - Setting to false stops conductors of negligible impedance being removed.
    • combineCommonImpedances - Setting to false stops connected conductors with the same impedance from being combined into a single conductor. Note: Without these network simplification operations enabled the final OpenDSS model may have extremely degraded performance or may not converge.
  • The Processor now adjusts the simplified network in processScenario to prepare it for the TimescaleDB-backed profile service in EWB:
    • Adds virtual UsagePoint to each PEU added in the scenario
    • Adjusts ratedPower of usage points to correspond to percentage scaling applied to the profile backing each one:
      • Loadshapes for DER are normalised in the input DB, so a 12kW PV system would have a ratedPower of 1200
      • Profiles for CPIs are absolute, so a ratedPower of 100 means no load scaling and 150 means 150% load scaling.
  • Added useSpanLevelThreshold to CommonImpedancceCollapsor so it can be used to make AcLineSegment collapse base off designRating and not ratedCurrent.
  • Added ImpedanceNormalizer to NetworkSimplifier so AcLineSegment with similar PLSI can be normalized to have same PLSI when value differences are within a threshold.
  • Added ratingThreshold to CommonImpedanceCollapsor so it can be used to loosen up AcLineSegment collapse throughdesignRating or ratedCurrent.
  • Added emergAmpScaling to ModelConfig so emergency amp rating of LineCode can be scaled base on user input.
  • Added support for translating generation equipment in the CIM model to PvSystems controlled by InvControl's in OpenDss depending on configuration in inverterControlConfig field of ModelConfig.

EnhancementsŌĆŗ

  • Added command line flag --write-out-sample-list to enable writing out the node level result sample points to disk if --output-models-path has also been provided.
  • Added better error logging during OpenDssToCim processing to help with debugging process.
  • Processor now uses the SyfDecoded DTO from commons instead of SyfJob, which serves the same function.
  • Added command line flag --forecast-equipment-commission-month-day to allow configuring the day of the year that will be used as the commissionDate for equipment added during forecast scenario generation.
  • PqvLoadFetcher now has the ability to fall back on to FixedTimeLoadFetcher during a calibration work package if no data was retrieved from the PQV table.

FixesŌĆŗ

  • Number of step of a fixed time model will now check the number of entry of the largest load profile in the model and use that as the step number.
  • Fix issue where ctPrimScalingFactor was used even when unset resulting in incorrectly configured OpenDSS models
  • Only create a virtual usage point once for each PowerElectronicsUnit when applying a scenario
  • The aggregateUsagePoints function in LoadUtils now correctly aggregates UsagePoint loads.
  • Fixed issue with conditions that determine if an isolation transformer should map to a REFCL transformer in the CIM to OpenDSS translation.
  • Fixed load aggregation for energy consumers with multiple usage points.

NotesŌĆŗ

  • Updated to com.zepben.commons and com.zepben.ewb v1.1.0
  • Updated to use EWB schema 63.
