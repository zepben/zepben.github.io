Network Fixer Algorithms

The Network fixer utilises load data to fix data inaccuracies in the network model, aiming to find unrealistic scenarios in the data that will cause load flow convergence to fail, and that can be fixed with sensible changes to the network model. See below for descriptions of the implemented fixing algorithms.

Overloading Consumer Fixer​

The overloading consumer fixer finds consumers that have peak load or generation (within the modelled time period) greater than the capacity of the transformer they are attached to by a configurable factor, and then reconfigures them to be HV consumers (attached above the transformer). The aim is to identify HV consumers that have been incorrectly connected as LV consumers, and resolve this connectivity.

As an example, if a single consumer X has a peak load of 100kW, but it's attached to a transformer with only a 25kVA rating, and we have configured a maxLoadTxRatio of 3, the consumer will get reconnected from below the transformer to above it (as 25*3 < 100), and then treated as a HV consumer.

This check will happen separately for load and generation, and the ratios can be configured with maxLoadTxRatio and maxLoadGenRatio for each respectively. By default we allow much larger ratios for generation, however it may make sense to tweak this depending on your circumstances.

Single Phase Load Fixer​

The single phase load fixer finds consumers that have a peak load (within the modelled time period) greater than a configured value (default 30kW), and upgrades them to three phase loads. The intent is to correct data inaccuracies where the number of phases reported for a consumer appears to be incorrect. By default we expect a 30kW load would not appear on a single phase consumer, so we upgrade them to three phase. This consists of tracing upstream to the distribution transformer and spreading 3 phases (ABCN) back to the transformer where possible.

An example would be if a single consumer has a 40kW peak load within the modelled year, but is only connected on B phase downstream of a 3 phase distribution transformer, it would be upgraded to ABC, and ABC would be spread from the consumer, to its service line, and upstream to the transformer or the first conducting equipment that is 3 phases.

Undersized Service Line Fixer​

The undersized service line fixer finds consumers that have a peak load (within the modelled time period) greater than the capacity of the service line of the consumer by some configured factor. The intent is to find service lines that have unrealistically low current ratings which would stop convergence, and upgrade them to sensible ratings. When a conductors rating is upgraded, we also then upgrade the impedances to a type in line with the new rating, utilising a pre-configured catalogue of rating and impedance data, and matching the phase configuration of the consumer.

As an example, if a single consumer X has a peak load of 15kW, but it's attached to a service line that will only handle 5kVA rating, and we have configured a maxLoadServiceLineRatio of 2, the service line will get upgraded to the next available rating in the catalogue that supports the load.

Undersized LV Line Fixer​

The undersized LV line fixer operates in the same way as the service line fixer, however acts on LV conductors upstream of the consumer that are not service lines. The same logic takes place, but using the configured maxLoadLvLineRatio, and will impact all LV conductors upstream of the consumer all the way to the distribution transformer.

As an example, if a single consumer X has a peak load of 15kW, but there are some LV conductors upstream of it that only handle a 5kVA rating, and we have configured a maxLoadLvLineRatio of 2, all of these LV conductors with a 5kVA rating will get upgraded to the next available rating in the catalogue that supports the load.
