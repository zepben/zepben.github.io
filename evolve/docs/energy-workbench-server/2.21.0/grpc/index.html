gRPC Services

Energy Workbench offers several gRPC services for accessing the underlying model hosted by the server. These interfaces are defined with scale and speed in mind, and allow consumption of large amounts of data from the server in an efficient manner. The EWB Java and Python SDKs provide consumer clients that do the heavy lifting of interacting with these gRPC services for you.

Currently there are 3 available services:

  1. NetworkConsumerService
  2. CustomerConsumerService
  3. DiagramConsumerService

Each service is described below. For examples of using these services, see the corresponding documentation in the available SDKs.

warning

Interacting directly with the gRPC services is not supported. Please use the SDKs and raise a support request if functionality is not present in the SDK.

Common across all services​

getIdentifiedObjects​

Returns a stream of IdentifiedObjects from the service, corresponding to the mRIDs provided in the request. This can be used for retrieving individual objects and will return a oneof set with the corresponding type.

NetworkConsumerService​

The NetworkConsumerService is responsible for retrieving network (e.g hierarchy, equipment, connectivity) data from the EWB server.

See relevant protobuf definitions here

getNetworkHierarchy​

Returns a response containing all hierarchy and EquipmentContainer components of the network. This will include:

  • GeographicalRegions
  • SubGeographicalRegions
  • Substations
  • Feeders
  • Circuits (if sub-transmission is available)
  • Loops (if sub-transmission is available)

getEquipmentForContainers​

Returns a stream of network related IdentifiedObjects that belong to the EquipmentContainer mRIDs provided in the request for the normal state of the network, as well as downstream/upstream containers if requested.

getCurrentEquipmentForFeeder​

Returns a stream of network related IdentifiedObjects that belong to the EquipmentContainer mRIDs provided in the request for the current state of the network, as well as downstream/upstream containers if requested.

getEquipmentForRestriction​

Returns a stream of IdentifiedObjects that belong to the OperationalRestriction mRID provided in the request.

getTerminalsForNode​

Returns a stream of Terminals that belong to the ConnectivityNode represented by the mRID provided in the request.

CustomerConsumerService​

The CustomerConsumerService is responsible for retrieving customer data from the EWB server.

getCustomersForContainer​

Returns a stream of customer related IdentifiedObjects that belong to the EquipmentContainer mRIDs provided in the request for the normal state of the network.

DiagramConsumerService​

getDiagramObjects​

Returns a stream of DiagramObjects related for the requested mRIDs provided in the request. Note this will call DiagramService.getDiagramObjects() to fetch the relevant DiagramObjects for the given mRID. See the documentation for that function for details.
