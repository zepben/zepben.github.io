Network Model

This page contains the schema definition for the network model database with filename format : <YYYY-MM-DD-network-model.sqlite> under a single date subdirectory.

Conventions#

All of the table definitions use the following conventions:

Key Fields#

Key fields are highlighted using the following convention:

primary_key Indicates this field is the primary key(or part of a composite key).

foreign_key Indicates this field is a foreign key into another table. Details of the relationship will be defined in the description of the field.

Field Units#

If a fields values should be entered using a defined unit, it will be specified in round brackets following the field definitions. All fields that define units do so because the software make assumptions about the values in these fields (e.g. to perform calculations).

Common Column Definitions#

Below you will find the schema description for table parts that are common amongst multiple tables.

Assets#

All tables that store assets will contain the following:

Connectivity Nodes#

Although not strictly a foreign key, the connectivity node fields used in both the Conductors and Node Terminals tables provide a linkage between each other. All asset terminals that use the same connectivity node will be connected using the wiring provided. The number of cores on the connectivity node matches the terminal with the largest number of cores.

Wiring#

The wiring fields are strings of integers that are 0 based indices which indicate the way the cores of the terminal are mapped on the connectivity node. The position in the string is the core number of the terminal, while the digit in the position is the core on the connectivity node.

Examples#

The following examples are for common use cases of connectivity node wiring.

Balanced Cores#

The most common scenario in a network will be when all terminals connected to the connectivity node have the same number of cores. In this case they will almost exclusively be wired straight through, meaning that in a 4 core connectivity node the wiring string will be 0123. This represents terminal core 0 mapping to connectivity node 0, 1 to 1, 2 to 2 and 3 to 3.

Uneven Cores#

The other scenario is when the number of cores do not match. We will take a look at a single phase drop off a backbone. In this case it is likely that you will have three conductor terminals connected to the connectivity node, two with 4 cores and one with 2 cores (the 4 core terminals will be wired as in the Balanced Cores example).

The following wirings make the assumption that a source has been applied to each of these cores such that ABCN are the phases of the cores 0123.

A B drop off would have a wiring of 13 to provide a phasing of BN, an AB drop off would have 01 to provide a phasing of AB.
