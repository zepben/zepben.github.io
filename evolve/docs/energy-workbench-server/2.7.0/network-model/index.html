CIM Model#

The EWB Server is composed around a domain model based on the 'Common Information Model' (CIM). The CIM is a very large standard that covers a huge amount of use cases. To make things more digestible, Evolve publishes its own CIM profile. CIM profiles are subsets of the whole CIM standard that dictates which parts of the model are in use. Evolve publishes its model at https://zepben.github.io/evolve/docs/cim/evolve/.

The Evolve CIM Profile has been implemented in the Evolve JVM SDK on which the EWB Server has been built. More information on can be found in the Evolve JVM SDK documentation.

If the Evolve profile doesn't contain a part of CIM that you require for your use case, you can request or propose a change to the model by starting a discussion at the Evolve GitHub discussions or by contacting Zepben directly at https://www.zepben.com/contact.
