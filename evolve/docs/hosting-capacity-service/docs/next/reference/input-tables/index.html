BESS Profiles​

Full Table: bess_profiles
Column NameData TypeConstraintsDescription
profile_idTEXTPRIMARY KEY, NOT NULLID of this BESS profiles configuration
nameTEXTNaNUser-friendly name to describe the profile. For display purposes to the end-user only.
use_typeTEXTNaNType of end use for BESS, primary use is illustrative for the user
optimisation_typeTEXTNaNType of control optimisation. Primary use is illustrative for the user
represents_yearINTEGERNaNYear represented by profile. Depending on the setting of "lock_year," only BESS_profiles with represents_year == scenario_year can be allocated.
feeder_codeTEXTNaNFeeder represented by profile. Depending on the setting of "lock_feeder," only BESS_profiles with feeder_code == scenario_feeder can be allocated.
intervals_per_yearINTEGERNaNNumber of intervals in the normalised_power array, where the time period is assumed to be one year.
demand_responseBOOLEANNaNFlag to indicate that the profile includes some response to pricing or demand reduction signal.
sub_geo_regionTEXTNaNGeographical region the PV_profile represents/is located within.
phasesINTEGERNOT NULLNumber of phases used to connect charger - note modeling assumes peak_capacity is balanced across available phases.
round_trip_efficiencyREALNaNRound-trip efficiency is the percentage of electricity put into storage that is later retrieved.
dedicated_inverterBOOLEANNaNUsed to indicate if the profile is representing a profile adjusted to account for shared use of inverter with PV system.
system_ageINTEGERNaNSystem age, used to indicate the amount of degradation built into the profile.
ac_capacityREALNaNNameplate kVA capacity of the inverter used to connect BESS to the grid.
energy_storage_capacityREALNaNNameplate kWh capacity of the BESS.
normalised_powerDOUBLE PRECISION[]NaNNormalized array of AC System Output (W) magnitudes. Intervals expected are defined by intervals_per_year.
