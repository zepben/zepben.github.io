1. Introduction to Calibration​

Accurate network model calibration is a prerequisite for reliable power system analysis and is particularly crucial for voltage regulation studies and determining unknown distribution transformer tap positions. The objective is to ensure that the power flow models generated are an accurate reflection of real-world system conditions, providing a validated baseline for further analysis.

For more information on why network calibration is important and the underlying concepts, see What is Network Model Calibration and why is it useful?

2. Calibration Methodology​

The calibration process integrates measured field data with the Hosting Capacity Modules OpenDSS power flow simulation environment through the following steps:

2.1. Data Ingestion and Preparation​

Required inputs (these should have been automated into your deployment during Hosting Capacity Module deployment):

  • Historic power quality data at the energy consumer level, which includes measurements of watts (P), vars (Q), and voltages (V) or alternatively amps, power factor, and voltages. This data is often referred to as PQV data. It is not necessary for this data to cover complete weeks, months, or years; rather, it should encompass a range of grid conditions, including minimum, maximum, and typical network loading periods. You may wish to consider running Model Calibration over various times to explore the impacts of different system conditions, for example, proving data for periods such as 7 am, 12 pm, 7 pm, and 3 am could be useful to explore periods of medium demand, high generation, high demand, and low demand respectively.
  • Feeder head voltages corresponding to the feeder identifier are used within the Energy Workbench. These are typically the phase-to-phase voltages measured at the medium voltage (MV) bus of the feeder's Zone Substation. Including this data helps eliminate uncertainty related to the upstream high voltage (HV) network and the modelled voltage regulation settings. These measurements are stored in the feeder_head_metrics table. For calibration, the average of the provided phase-to-phase voltages is used, based on the assumption of a balanced network at the feeder head.

2.2. Power Flow Model - Calibration 'Mode'​

When calibration work packages are run, the OpenDSS power-flow models are built to model a specific historic timestamp for which historic input data exists:

  • The net demand position of each usage point (NMI) in the network is modelled as a 'bi-directional' Load object within the model, which maps directly to the instantaneous 5-minute P and Q power data stored within the TimescaleDB time-series database.
  • The feeder head tap changer representation is replaced with an ideal voltage source, its value set to the measured actuals from the feeder_head_metrics table.
  • measurement_zone objects are strategically allocated to the feeder head and the usage points at each customer connection (NMI) to provide voltage measurements that align with the location of real-world smart meter voltage actuals, facilitating comparison between modelled and measured values.
note

In Calibration Mode, the OpenDSS model has measurement_zone objects placed at each NMI, unlike in the Hosting Capacity Module's standard power flow model, which only has measurement_zone objects at the feeder head. This more granular appraoch allows for a direct comparison of modelled voltages against real-world measurements at each energy consumer location. This is more computationally intensive and generates more data to store, but is necessary for the calibration process to infer off-load tap positions and validate the model's accuracy, and given calibraion is done only for a single timestep, the computational overhead is acceptable.

2.3. Load Flow Execution​

  • A load flow analysis is executed on the updated OpenDSS power flow model for each defined time case within the calibration work package.
  • Results are streamed to the specific energy_meter_voltages_raw table, where the conducting_equipment_mrid lines up with the energyconsumer_mrid (supply point) and can then be used as part of the automated analysis to infer off-load tap changer positions and 'calibrate' the model. This is unlike the standard HCM, which has energy_meter_voltages_raw off by default, as it would generate too much data for the standard HCM use case.

2.4. Off-Load Tap Position Determination​

  • For each energyconsumer location, a voltage difference (ΔV) is calculated based on the difference between the average of the available real-world measured phase voltages and the average of the modelled phase voltages.

  • For each transfomer, an overall average is calcuated, based on the average of all voltage deltas from the energy consumers fed by that transformer.

  • This transfomer voltage delta is then used in combination with the associated transformer's current tap position and tap size to determine an adjustment, and a new tap postion is returned.

    note

    The tap adjustment is calculated using current tap postions if known, and if not known, then assuming a value of 4 out of 7 taps, which is a common default for many distribution transformers. This is because the tap position is often unknown in most Distribution Network Service Provider (DNSP) network datasets. It assumes a tap size of 2.5% per tap, which is a common value for many distribution transformers. This is currently hardcoded.

  • The resulting tap positions are stored in the calibrated_taps table, where various 'sets' of off-load tap positions can be determined for different calibration studies. These sets of off-load tap positions can then be selected by users on demand as part of the work package configuration when running future time-series modelling work packages using the transformerTapSettings parameter.

2.5. Model Validation and Discrepancy Analysis​

  • Similar to the Off-Load Tap Position Determination flow, this workflow can be used to compare the voltages (V) obtained from the load flow simulation with the actual measured voltages (i.e., the raw PQV data) at energyconsumer locations. The voltage difference (ΔV) is calculated for these points: ΔV = Actual Raw Voltages - Load Flow Voltages.
  • Review of these energyconsumer level voltage delta results, without seeking to adjust off-load tap positions, provides an ability to measure the accuracy of the model and develop a view for the range of model uncertainty for various network demand scenarios.
  • Further analysis of the relationship between demand and voltage error can provide insights into the uncertainty of the impedance model; however, no built-in tools are included for this analysis.
  • The results are stored in the calibration_meter_results table, which contains detailed voltage delta calculations and associated metadata. TODO: Verify if additional metadata details need to be included.
  • Processed results are stored within the calibration_meter_results table, which contains voltage delta calculations and flagged instances for further analysis.
  • Instances where ΔV exceeds a predefined threshold (e.g., 1.5%) are flagged. (?)
  • A list of transformers and associated energyconsumers exhibiting these significant voltage deviations is generated for review and further investigation. This step is crucial for identifying potential model inaccuracies or data anomalies.
  • For both the model validation use case and the off-load tap position use case, users are able to configure thresholds that determine if a measured actual is valid and should be included, or if a simulated voltage is valid and should be included in the calculation of voltage deltas. For example, users can specify a threshold of ±1.5% for voltage deviations, meaning that any measured or simulated voltage outside this range will be excluded from the analysis. These thresholds are typically defined in the calibration work package configuration file under the voltage_threshold parameter, allowing flexibility to adapt to different network conditions and study requirements.
