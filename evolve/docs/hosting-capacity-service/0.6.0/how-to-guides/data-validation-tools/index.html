Scenario Configuration Validator for Hosting Capacity Model (HCM)

Overview​

The Scenario Configuration Validator is a data quality assurance tool for the Hosting Capacity Model (HCM) system. It validates the scenario_configuration input table against all related tables to ensure data integrity before running HCM work packages.

Purpose​

The validator prevents runtime errors in HCM analyses by catching configuration issues such as:

  • Invalid scenario names or duplicate IDs
  • References to non-existent allocation IDs or forecast scenarios
  • Mismatched forecast levels between configuration and actual data
  • Incorrect probability weightings in allocation strategies
  • Missing profile references

How It Works​

  1. Connects to the input database via SSH tunnel
  2. Loads all reference data (allocation IDs, forecast scenarios, profile IDs) into memory
  3. Validates each scenario configuration against 8 different validation types
  4. Outputs detailed results to timestamped CSV file with console summary

Output Files​

  • Primary output: scenario_config_validation_YYYY-MM-DD_HH-MM.csv
  • Contains: One row per scenario with all validation results and code counts

Interpreting Results​

Successful Validation​

  • All validation columns show PASS
  • overall_status shows PASS
  • Code counts show expected network coverage

Failed Validation​

  • Failed validation columns show specific error messages
  • overall_status shows FAIL
  • Console output provides detailed failure summary
