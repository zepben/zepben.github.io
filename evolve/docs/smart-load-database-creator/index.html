Smart Load Enabled SINCAL Database

If you have created the model from a GIS export, you will need to convert each data model to a smart load enabled model.

If you have created the model from the SINCAL export utility (ie using the DMS data in schematic form, you do not need to use the Smart Load Database Creator. It will be ready to use immediately.

info

If you use the Save as command from within SINCAL to copy the database, it will not update the model’s .ini file, which means you will need to run the conversion tool against the newly created model again.

The smart load maker executable is db-smart-load-maker.exe. The screen capture below illustrates the application in action.

Its use is simple – Select one or more .sin files to process, and press the Process button. This will update the selected databases, providing a log with any exceptions encountered.

caution

The element table in SINCAL, which holds the load data names, only has space for 50 characters, but some of the names exported by the GIS can be longer than this. In this case, the name is truncated in the element table during the import process from the GIS.

The GIS ID is the last item in the name. The database converter strips this ID from the name and the smart load plugin uses this ID to ask for load data from the server.

If this ID is truncated, because the name originally written into the element table was too long, no load will be found for that load element and the value entered into the load element will be used.

This situation will be logged, and a string written into the MasterResource table in the form “ERROR_<Truncated Name>_n”.

You can fix the error by manually replacing this with the real GIS ID.
