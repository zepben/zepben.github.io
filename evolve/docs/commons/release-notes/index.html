[1.3.0]​

Breaking Changes​

  • The RabbitMqClient now throws an InitialConnectionException that wraps the underlying reason for the failed connection, rather than the raw exception

New Features​

  • Add an optional RabbitMQ exception handler that can try recover a connection a finite number of times, rather than retrying forever
  • Add a tryConnect function that tries to connect to RabbitMQ a number of times before failing
  • Added an overload to RabbitMqClient.publish that allows raw byte arrays to be message bodies

Enhancements​

  • Automatically emit RabbitMQ client metrics if OpenTelemetry metrics are enabled.
  • Improved the default change set conflict processor to allow duplicate objects with changes to connectivity and container membership to modify the base model.
    • The RabbitMqClient now retries initial connections 3 times before failing, rather than no retries previously
  • Add additional optional argument passthroughs to the RabbitMqClient
  • Added generic removal extension functions for LvSubstation, HvCustomer, Site, and AcLineSegmentPhase.

Fixes​

  • CsvCharacterStreamWriter.writeRecord will now correctly coerce null values into empty strings, rather than incorrectly treating it as a missing column.
  • Adjust function signature on RabbitMqClient.createQueue to allow kotlin maps to be passed in. Previous signature cause type checking issues

Notes​

  • Updated ewb-sdk from v1.2.0 to v1.3.0.

[1.2.0]​

Breaking Changes​

  • Removed Column.Nullable.NONE in favour of setting the column nullability explicitly. Nullability left unspecified should default to nullable in every ANSI-compliant implementation of SQL, so you likely can replace any use with Column.Nullable.NULL.

  • NegligibleImpedanceCollapser minLineResistanceOhms and minLineReactanceOhms have been replaced by minHvConductorImpedance and minLvConductorImpedance, which measure the hypotenuse between r and x to determine whether a conductor should be collapsed.

  • Added ability to provide reactive load and generation in energy profiles, requiring additional arguments for the following functions. In most cases, the additional arguments (kvarIn and kvarOut) come after kwIn and kwOut, and can be set to null if no suitable data is available.

    • new EnergyProfile() (protected constructor)
    • new EnergyProfileStat() (kvarIn, kvarOut, and kvarNet come after the existing 3 stat parameters)
    • new AggregatedEnergyReadings() (pf is turned into a read-only property that is inferred from the provided real and reactive values)
    • EnergyProfileFactory.create() (functional interface)

    Overloads for EnergyProfile.of() and EnergyProfile.ofCacheable() have been added to support the new reactive components, but their original method signatures are still supported and produce energy profiles without reactive load. This change also adds 3 more fields to the EnergyReadings interface: kvarIn, kvarOut, and kvarNet.

  • Added nullability to energy profile values to represent missing data with null instead of NaN:

    • com.zepben.commons.energy.model:

      • DoubleArrayView now represents an array view of Double?s rather than primitive doubles, and so does its inheritors:

        • Channel
        • DoubleChannel
        • FloatChannel
        • NullsChannel
        • ZeroedChannel
        • Readings
        • MultiChannelReadings
        • MissingReadings
        • ZeroedReadings

        Constructors and factory functions for these classes that used to take one or more doubles now take nullable Doubles, breaking usages in Java that exploit automatic type widening e.g. Channel.of(1, 2, 3) must now be written as Channel.of(1., 2., 3.). This also necessitates using Double[] and Float[] instead of double[] and float[] when passing in arrays of values, so in Kotlin doubleArrayOf(...) and floatArrayOf(...) become arrayOf<Double?>(...) and arrayOf<Float?>() respectively.

      • DoubleArrayView.max() and DoubleArrayView.min() now return null when no max/min can be computed due to a lack of nonnull values (including cases where the DoubleArrayView is empty i.e. length() == 0), instead of throwing an IllegalStateException.

      • EnergyProfile is made of 4 Readings (kwIn, kwOut, kvarIn, kvarNet), with each potentially having null values due to the above change.

      • EnergyProfileStat uses nulls to denote stats that could not be calculated due to missing data, e.g. EnergyProfileStat.ofMax(profile).kwOut() being null means the profile had no real generation to get a maximum for.

      • MissingReadings now extends Readings instead of ZeroedReadings, since it represents an array of null readings instead of 0 readings.

      • ZeroedChannel now throws an IndexOutOfBoundsException instead of an IllegalArgumentException when calling get(i) for i < 0 and i >= length(). This matches the behaviour of DoubleChannel and FloatChannel. The same applies to the new NullsChannel class.

      Note that for DoubleChannel, FloatChannel, and EnergyProfileStat, NaN is used internally to denote nulls, breaking preservation of NaN values: If you instantiate one of these classes specifying NaN for any of the Double values, it will be returned as null in the getter function (get(Int) for DoubleChannel/FloatChannel, and kwIn(), kwOut(), etc. for EnergyProfileStat).

    • com.zepben.commons.load.queryengine:

      • AggregatedEnergyReadings uses nulls for each of its 4 components to denote that no nonnull values were provided. Aggregation now ignores missing values, so addKwIn(1.0); addKwIn(1.5); addKwIn(null) results in a kwIn of 2.5 instead of null. This is different from before where missing values were represented with NaNs and they were preferred instead of ignored, so addKwIn(1.0); addKwIn(1.5); addKwIn(NaN) resulted in a kwIn of NaN instead of 2.5.
      • AggregatedLoadProfile, due to above change, may have nulls in its readings to denote data missing across the provided profiles. Like AggregatedEnergyReadings, existing (nonnull) data is preferred over missing (null) data during aggregation, as opposed to before where missing (NaN) data is preferred over existing (non-NaN) data during aggregation.
      • EnergyReadings: All components (kwIn, kwOut, kvarIn, kvarOut, kwNet, kvarNet, kvaNet, pf) are now nullable.

  • Channel.EMPTY_CHANNEL is replaced with Channel.emptyChannel(), which is a NullsChannel of length 0.

  • Readings.EMPTY_READINGS is replaced with Readings.emptyReadings(), which is a MissingReadings of length 0.

  • CollectionDifference is now abstract, with two descendants (ValueCollectionDifference and ObjectCollectionDifference) to allow detection of differences in the object itself, and in having different references to other objects.

  • ratedPower is now set to the per-0.001% value of load scaling for EnergyConsumers' usage points, and max charging/discharging rate in 0.01VA for PowerElectronicsUnits' usage points.

New Features​

  • Added getContainerExecution to ContainerTemplate to allow retrieval of the ContainerExecution object for a running container.
  • Added byteArrayToRemote to StorageBackend to allow writing a byte array directly to remote storage.
  • Added Kotlinx serializers for the following Java types:
    • java.nio.file.Path
    • java.time.LocalDate (ISO_DATE)
    • java.time.LocalTime (ISO_TIME)
    • java.time.MonthDay
    • java.util.UUID
    • ch.qos.logback.classic.Level
  • Added support for specifying container resource requirements in ContainerTemplate.start, allowing CPU and memory requests and limits to be set. Currently supported in Docker and Kubernetes runtimes.
  • Added hosting capacity notification messages
  • RabbitMQ implementation to consume and publish notification messages.
  • Added postgres support for Column.Type.
  • Added the following new Column.Type values:
    • UUID
    • TIMESTAMP
    • BYTES
  • Added support for variants to EwbDataFilePaths. You can now provide a variant name to EwbDataFilePaths.resolve to generate a variant filename, and call getAvailableVariantsFor to get the list of available variants for a specified date.
  • Added HvCustomerMover processor for moving HV customers to the HV side of their distribution transformer.
  • Added support for specifying reactive values for energy readings and profiles.
    • Also added nullability for all 4 components (kwIn, kwOut, kvarIn, kvarNet) to represent missing data.
  • Added utility class FloatingPointRounder, which contains static methods for rounding floating point numbers to a given number of digits in a performant way. This class was originally in energy-workbench-server and ewb-network-routes under the name FloatingPointTruncator.
  • Added RemovalExtensions which contains extensions to NetworkService, CustomerService, and DiagramService for cleanly removing IdentifiedObjects and their associations from the service. These functions are currently experimental, however users should prefer adding to them rather than creating their own removal functionality.
  • Allow KubernetesJobTemplate environment variable references to be optional. Defaults to existing behaviour of false
  • You can now opt out of the post read processing in the database readers using the optional preformAfterReadProcessing parameter.
  • Added change set processing in com.zepben.commons.changeset:
    • ChangeSetProcessor - Main entry point via the applyChangeSet extensions for NetwrokService, DiagramService and CustomerService, allowing you to merge the contents of a change set service.
    • ConflictHandler - A customisable set of callbacks responsible for assessing and handling conflicts and determining the outcome of the conflict.
    • ConflictIssues - Issue tracker for the conflict handler. Can be composed into an applications issue tracking.
  • Added GeoJsonFeature class which allows the conversion of PowerSystemResources to GeoJson
  • Added two new helper functions to assist with providing an mRID for IdentifiedObject instances:
    • com.zepben.commons.utils.nextTerminalMrid extension for ConductingEquipment, which will generate an ID for the next terminal in the form of "<mrid>-t#".
    • com.zepben.commons.utils.test.generateId which can be used by tests where known mRID's are not important.

Enhancements​

  • Added addFromPb methods to accept the top level protobuf messages for CustomerIdentifiedObject, DiagramIdentifiedObject and NetworkIdentifiedObject.
  • AggregatedLoadProfile no longer assumes a constant load factor of 0.9.
  • Updated HvCustomerMover to allow specifying shouldRemoveLv which as well as moving EnergyConsumers above the transformer, will also remove the associated LV network and the transformer.
  • Stopped using generated mRID's for objects. All objects created by processors in commons will now use repeatable mRID's.

Fixes​

  • NegligibleImpedanceCollapser will now determine whether to collapse based on impedance, rather than independently checking r and x values.
  • NegligibleImpedanceCollapser will now also avoid collapsing conductors connected to an EnergyConsumer. If all conductors between the energy consumer and the next branch or uncollapsible equipment are of negligible impedance, they will be replaced with a 0.001m length conductor of minimum allowed impedance.
  • ByDateBlobEnergyProfileWriter.write() no longer writes IDs for profiles with no readings (MissingReadings in all components).
  • The XyPhaseProcessor now correctly finds hints when available at the point of phase splitting, rather than using the phase hints from the upstream.
  • Do not delete Kubernetes jobs when stop() is triggered, to allow for other asynchronous Kubernetes jobs to take place before cleanup.
  • PostgresLoadDataFetcher ignores virtual usage points when requesting load data for UsagePoints or EnergyConsumers. A LoadData with null load is returned if all usage points for the request are virtual.

Notes​

  • Amend spec.ttlSecondsAfterFinished on Kubernetes jobs to be 15 minutes

[1.1.0]​

Breaking Changes​

  • Added required constructor param issues: UsagePointAggregationMappingsExtractorIssues to UsagePointAggregationMappingsExtractor.
  • Removed extension function NetworkService.applyLoadScalingFactor(). This functionality has been moved into NetworkService.applyScenarioYear().
  • Changed signature of NetworkService.applyScenarioYear():
    • scenario, year, and feederMRID have been replaced with syf: SyfDecoded. This was done to avoid having to add a new parameter for each config field that needed to be passed through from the SYF message.
    • A loadDataFetcher: LoadDataFetcher must now be provided. This is used for fetching the peak load and generation when calculating load scaling factors.
    • Added parameters winterSeasonStart: MonthDay and winterSeasonEnd: MonthDay, which are used in calculating load scaling factors. These default to the 1st of June and the 31st of August respectively, which are sensible values for Australia.
    • Added commissionDate which will be applied to new equipment added while applying the scenario.
  • Discrepancies in the v61 upgrade scripts and the schema creation objects have been fixed. This includes correcting the types of some columns, changing the nullability of some columns and adding missing indexes.
  • InputFetcher.fetchForecastTotalDemand() now returns a Double? instead of a Double, with null indicating no forecasted demand could be found for the specified feeder, year, scenario, etc.
  • Timestamp.toInstant() and Instant.toTimestamp() have been removed from the public API. If you were using them, you will need to copy the implementation.
  • The following gRPC fields have been modified to support nulls (they were missed in v1.0.0):
    • Document.createdDateTime
    • Equipment.commissionedDate
    • MeasurementValue.timeStamp
    • RelayInfo.curveSetting
    • RelayInfo.recloseFast
  • NetworkCimReader is now an AutoClosable to clean up local caches used when loading the database. Failure to close the reader simply means the unused memory will be kept indefinitely, rather than being released back to the server, but you may get code analysis errors due to this change.

New Features​

  • Added configurable limit to the number of branches UsagePointAggregationMappingsExtractor creates in each feeder trace before giving up and moving to the next feeder.
  • Added interventionConfig to SyfDecoded and added sensible defaults for most parameters.
  • Added the following new CIM classes:
    • DateTimeInterval, interval between two date and time points, where the interval includes the start time but excludes end time.
    • ElectronicAddress, electronic address information.
    • TelephoneNumber, telephone number.
  • Added the following new CIM extension classes:
    • ContactDetails, the details required to contact a person or company. These can be accessed/used via a UsagePoint.
    • DirectionalCurrentRelay, a directional current relay is a type of protective relay used in electrical power systems to detect the direction of current flow and operate only when the current exceeds a certain threshold in a specified direction.
  • Added new CIM extension enums:
    • ContactMethodType
    • PolarizingQuantityType
  • ContainerTemplate now support passing in a map of tags to be applied to the resulting resource. Currently only supported by KubernetesJobTemplate
  • Added optional inverterControlConfig object field to ModelConfig for configuring the translation of generation equipment into OpenDss elements(Advanced PV).

Enhancements​

  • The phase consistency algorithms now support repairing upstream paths from LV transformers (e.g. regulators).
  • Missing demand forecasts are no longer treated as a malformed scenario, but rather defaults the load scaling for the feeder in question to 100%.
  • BaseService.contains has been marked as an operator, allowing Kotlin to use in against a service. It has also been expanded to support objects in addition to mRIDs.
  • Agreement now supports validityInterval, the date and time interval the agreement is valid (from going into effect to termination).
  • StreetDetail now supports extension buildingNumber, the number of the building.
  • TownDetail now supports country, the name of the country.
  • The compareUnorderedValueCollection methods in BaseServiceComparator and PropertyOmpareres have been relaxed to take any Colleciton rather than only accepting a List.
  • PqvLoadFetcher can now fallback on to FixedTimeLoadFetcher if no entry was found in the PQV table.
  • KubernetesJobTemplate jobs are now labelled with a standard app label both at the pod level and the job level
  • Scenario generation now accepts a commission date for newly added equipment.

Fixes​

  • The support for repairing upstream paths from LV transformers also resolves a possible exception when back-feeding HV via connected LV transformers, which is almost certainly caused by errors or misalignment in the source data.
  • From 1.0.1
    • Fixed bug that would cause AzureBlobEwbDataFileUploader to fail to upload to blob storage.
    • Fixed bug where redundant voltage levels were created for equipment that was connected to nothing.
    • The Terminal extension function setSourceDirection now accepts a nullable SourceDirection as input and the SourceDirectionAssigner's clear method sets all Terminal's numDiagramObjects to null instead of 0.
    • AzureBlobEwbDataFileUploader now overrides instead of throwing BlobAlreadyExists error when blob object is already present.
  • Only create virtual peu usage points once
  • Fixed a bug in the LV feeder assignment when processing sites in the current state of the network. This prevented LV feeders on switches (rather than the transformer) from being assigned to the current state energising feeder.
  • Fixed case where UsagePointAggregationMappingsExtractor considers paths that backtrace across established edge
  • UsagePointAggregationMappingsExtractor now handles cycles in graphs gracefully by skipping the feeder it was built for.
  • Indexes have been fixed in the database schema.
  • CurrentStateEvent.timestamp now correctly clears the timestamp if you have a null timestamp.
  • The aggregateUsagePoints function in LoadUtils now correctly aggregates UsagePoint loads.
  • KubernetesJobTemplate containers are now named with the correct name to allow for better log correlation

Notes​

  • None.

[0.20.0]​

Network Verifier Lib

Breaking Changes​

  • You can no longer register a StepActionWithContextValue with Traversal.ifStopping or Traversal.ifNotStopping, you must use Traversal.addStepCondition instead.
  • Added a new debugLogging and name parameters to the constructor of the following traces. The helper functions in Tracing also have these parameters, which defaults to null and networkTrace, meaning anyone using these wrappers will be unaffected by the change:
    • AssignToFeeders
    • AssignToLvFeeders
    • ClearDirection
    • FindSwerEquipment
    • PhaseInferrer
    • RemovePhases
    • SetDirection
    • SetPhases
  • NetworkStateOperators has a new abstract description. If you are creating custom operators you will need to add it.

New Features​

  • You can now pass a logger to all Tracing methods and TestNetworkBuilder.build to enable debug logging for the traces it runs. The debug logging will include the results of all queue and stop condition checks, and each item that is stepped on.

Enhancements​

  • NetworkTrace now supports starting from a known NetworkTraceStep.Path. This allows you to force a trace to start in a particular direction, or to continue a follow-up trace from a detected stop point.
  • Support disabling certificate verification in SDK connection helpers

Fixes​

  • Phases are now correctly assigned to the LV side of an LV2 transformer that is in parallel with a previously energised LV1 transformer.

Notes​

  • None.

[0.16.0]​

Breaking Changes​

  • Included from evolve-sdk-jvm 0.23.0:
    • Updated to latest evolve-grpc major version.
    • Removed unused AuthType enum.
    • Removed unused kotlinx-serialization-json dependency.
    • Updated to latest ewb-conn, and hence the signature of these helper functions have changed:
      • Connect.connectWithSecret:
        • issuerDomain has been renamed to issuer;
      • Connect.connectWithPassword:
        • issuerDomain has been renamed to issuer;
    • Renamed TablePowerElectronicsUnit to TablePowerElectronicsUnits.
    • CIM object removal functions no longer support null. e.g. You must pass a valid Terminal to ConductingEquipment.removeTerminal rather than a nullable object.
    • DiagramObject.getPoint no longer throws an IndexOutOfRange exception for an invalid sequence number, and returns null to match other functions of this type.
    • Removed the PowerTransformer.getRating overload which took a rating value. You can still get a rating via its TransformerCoolingType.
    • Removed PowerTransformer.forEachRating which looped over the collection with an index that made no sense. Please loop over PowerTransformer.sRatings instead.
    • Equipment to EquipmentContainer links for LV feeders are no longer written to the database, they should never have been.
    • Refactored EwbDataFilePaths:
    • The EwbDataFilePaths class has been refactored into an interface to enhance flexibility and abstraction.
    • A new class, LocalEwbDataFilePaths, has been introduced to specifically handle the resolution of database paths for the local file system.
  • Added OpenDssReportBatch and a new failure OpenDSS report type to the hosting capacity API.
  • Switch.ratedCurrent has been converted to a double (used to be an integer). Type safe languages will need to be updated to support floating point arithmatic/syntax.
  • Deprecated TracedPhases, however the internal constructor property has been removed. Terminal.normalPhases and Terminal.currentPhases should be used instead of Terminal.tracedPhases going forward.
  • JWTAuthoriser.authorise no longer accepts a permissions claims field, instead it will attempt to retrieve claims from the "permissions" field if it exists in the token, or the "roles" field if the "permissions" field doesn't exist.
  • JWTAuthenticator has a new signature to accept a list of trusted domains rather than a single domain, and a JWTMultiIssuerVerifierBuilder rather than a UrlJwkProvider.
  • Auth0AuthHandler has a new signature and no longer accepts a permissionsField to pass onto JWTAuthoriser.authorise. (See above change to JWTAuthoriser.authorise)
  • AuthRoute.routeFactory has a new signature. Now accepts a list of TrustedIssuer's in place of a urlJwkProvider and issuer.
  • Removed obsolete SwitchStateClient and corresponding SwitchStateUpdate which only communicated with a server implementation that logged the functionality was not implemented.

New Features​

  • Included from evolve-sdk-jvm 0.23.0:
    • A file named after the ID of an ingestion job is now created when running MetricsDatabaseWriter.save(). For this feature to take effect, a modelPath must be provided when constructing the MetricsDatabaseWriter.
    • You can now remove the following by index:
      • PositionPoint from a Location.
      • DiagramObjectPoint from a DiagramObject.
      • RelaySetting from a ProtectionRelayFunction.
    • Data Model change:
      • Add phaseCode variable to UsagePoint
      • Added new classes:
        • Curve
        • CurveData
        • EarthFaultCompensator
        • GroundingImpedance
        • PetersenCoil
        • ReactiveCapabilityCurve
        • RotatingMachine
        • SynchronousMachine
  • Updated grpc to support InterventionConfig and initial implementation of NetworkState.
  • JWTAuthenticator now supports authenticating tokens from multiple different issues via the use of JWTMultiIssuerVerifierBuilder.
  • JWTMultiIssuerVerifierBuilder will create a JWTVerifier based on the iss (issuer) of the token provided to getVerifier(). The returned JWTVerifier will also validate that the audience claim matches the requiredAudience supplied to the JWTMultiIssuerVerifierBuilder.
  • TrustedIssuer now supports lazy fetching of TrustedIssuer.providerDetails by accepting a lambda that takes an issuer domain and returns a ProviderDetails.

Enhancements​

  • Included from evolve-sdk-jvm 0.23.0:
    • Changed NetworkContainerMetrics to a delegate type to assist in writing metrics creators:
      • NetworkContainerMetrics::plus(key: String, amount: Number): Increases a metric by a certain value. If the metric doesn't exist yet, it is automatically created and set to zero before being increased. A negative value may be used for amount to decrease the metric.
      • NetworkContainerMetrics::inc(key: String): Equivalent to NetworkContainerMetrics.plus(key, 1.0)
      • NetworkContainerMetrics::set(key: String, value: Int): Allows setting a metric using an integer rather than a double-precision float: 
        metrics[TotalNetworkContainer]["metric-name"] = 3
    • GrpcChannelBuilder tests the connectivity of newly created channels before returning them. This is done by calling getMetadata() against all known services. The channel is returned after the first successful response. Any connectivity errors will be propagated to the user. If no connectivity errors are encountered but no successful responses is received from the known services, a GrpcConnectionException is thrown.

Fixes​

  • Update superpom for dokka plugin upgrade so maven central deploy works again.

Notes​

  • Upgraded to support ewb-sdk-jvm 0.23.0

  • Updated to latest SDK - see SDK release notes for versions 0.15.0 and 0.16.0 for changes.
