[1.1.0]​

Breaking Changes​

  • Added required constructor param issues: UsagePointAggregationMappingsExtractorIssues to UsagePointAggregationMappingsExtractor.
  • Removed extension function NetworkService.applyLoadScalingFactor(). This functionality has been moved into NetworkService.applyScenarioYear().
  • Changed signature of NetworkService.applyScenarioYear():
    • scenario, year, and feederMRID have been replaced with syf: SyfDecoded. This was done to avoid having to add a new parameter for each config field that needed to be passed through from the SYF message.
    • A loadDataFetcher: LoadDataFetcher must now be provided. This is used for fetching the peak load and generation when calculating load scaling factors.
    • Added parameters winterSeasonStart: MonthDay and winterSeasonEnd: MonthDay, which are used in calculating load scaling factors. These default to the 1st of June and the 31st of August respectively, which are sensible values for Australia.
    • Added commissionDate which will be applied to new equipment added while applying the scenario.
  • Discrepancies in the v61 upgrade scripts and the schema creation objects have been fixed. This includes correcting the types of some columns, changing the nullability of some columns and adding missing indexes.
  • InputFetcher.fetchForecastTotalDemand() now returns a Double? instead of a Double, with null indicating no forecasted demand could be found for the specified feeder, year, scenario, etc.
  • Timestamp.toInstant() and Instant.toTimestamp() have been removed from the public API. If you were using them, you will need to copy the implementation.
  • The following gRPC fields have been modified to support nulls (they were missed in v1.0.0):
    • Document.createdDateTime
    • Equipment.commissionedDate
    • MeasurementValue.timeStamp
    • RelayInfo.curveSetting
    • RelayInfo.recloseFast
  • NetworkCimReader is now an AutoClosable to clean up local caches used when loading the database. Failure to close the reader simply means the unused memory will be kept indefinitely, rather than being released back to the server, but you may get code analysis errors due to this change.

New Features​

  • Added configurable limit to the number of branches UsagePointAggregationMappingsExtractor creates in each feeder trace before giving up and moving to the next feeder.
  • Added interventionConfig to SyfDecoded and added sensible defaults for most parameters.
  • Added the following new CIM classes:
    • DateTimeInterval, interval between two date and time points, where the interval includes the start time but excludes end time.
    • ElectronicAddress, electronic address information.
    • TelephoneNumber, telephone number.
  • Added the following new CIM extension classes:
    • ContactDetails, the details required to contact a person or company. These can be accessed/used via a UsagePoint.
    • DirectionalCurrentRelay, a directional current relay is a type of protective relay used in electrical power systems to detect the direction of current flow and operate only when the current exceeds a certain threshold in a specified direction.
  • Added new CIM extension enums:
    • ContactMethodType
    • PolarizingQuantityType
  • ContainerTemplate now support passing in a map of tags to be applied to the resulting resource. Currently only supported by KubernetesJobTemplate
  • Added optional inverterControlConfig object field to ModelConfig for configuring the translation of generation equipment into OpenDss elements(Advanced PV).

Enhancements​

  • The phase consistency algorithms now support repairing upstream paths from LV transformers (e.g. regulators).
  • Missing demand forecasts are no longer treated as a malformed scenario, but rather defaults the load scaling for the feeder in question to 100%.
  • BaseService.contains has been marked as an operator, allowing Kotlin to use in against a service. It has also been expanded to support objects in addition to mRIDs.
  • Agreement now supports validityInterval, the date and time interval the agreement is valid (from going into effect to termination).
  • StreetDetail now supports extension buildingNumber, the number of the building.
  • TownDetail now supports country, the name of the country.
  • The compareUnorderedValueCollection methods in BaseServiceComparator and PropertyOmpareres have been relaxed to take any Colleciton rather than only accepting a List.
  • PqvLoadFetcher can now fallback on to FixedTimeLoadFetcher if no entry was found in the PQV table.
  • KubernetesJobTemplate jobs are now labelled with a standard app label both at the pod level and the job level
  • Scenario generation now accepts a commission date for newly added equipment.

Fixes​

  • The support for repairing upstream paths from LV transformers also resolves a possible exception when back-feeding HV via connected LV transformers, which is almost certainly caused by errors or misalignment in the source data.
  • From 1.0.1
    • Fixed bug that would cause AzureBlobEwbDataFileUploader to fail to upload to blob storage.
    • Fixed bug where redundant voltage levels were created for equipment that was connected to nothing.
    • The Terminal extension function setSourceDirection now accepts a nullable SourceDirection as input and the SourceDirectionAssigner's clear method sets all Terminal's numDiagramObjects to null instead of 0.
    • AzureBlobEwbDataFileUploader now overrides instead of throwing BlobAlreadyExists error when blob object is already present.
  • Only create virtual peu usage points once
  • Fixed a bug in the LV feeder assignment when processing sites in the current state of the network. This prevented LV feeders on switches (rather than the transformer) from being assigned to the current state energising feeder.
  • Fixed case where UsagePointAggregationMappingsExtractor considers paths that backtrace across established edge
  • UsagePointAggregationMappingsExtractor now handles cycles in graphs gracefully by skipping the feeder it was built for.
  • Indexes have been fixed in the database schema.
  • CurrentStateEvent.timestamp now correctly clears the timestamp if you have a null timestamp.
  • The aggregateUsagePoints function in LoadUtils now correctly aggregates UsagePoint loads.
  • KubernetesJobTemplate containers are now named with the correct name to allow for better log correlation

New Features​

  • Added AggregateUsagePointCreator, which can create usage points at arbitrary points in the network for the purpose of load aggregation. Methods are added for common use cases, such as createDistributionTransformerAUPs() and createFeederHeadAUPs().
  • Added CpiUsagePointIngestor, which extracts the 1-to-1 mapping from connection point identifiers (CPIs) to the mRID of the UsagePoint holding the CPI as a name. Note that this does not update the cpi_aggregations table.
    • The constructor requires an instance of CpiUsagePointExtractor, which extracts mappings from the network, and an instance of CpiUsagePointWriter, which writes mappings to the load database.
  • Added UsagePointAggregationMappingsIngestor, which extracts the mappings from UsagePoint to aggregate UsagePoint from a network and writes it to the postgres load database. Note that this does not update the usage_point_aggregations table.
    • The constructor requires an instance of UsagePointAggregationMappingsExtractor, which extracts mappings from the network, and an instance of UsagePointAggregationMappingsWriter, which writes mappings to the load database.
  • Added concrete implementation AzureBlobEwbDataFileUploader, which handles the specific logic for uploading files to Azure Blob.
  • Added DerLoadshapeFetcher, which fetches and caches DER loadshapes from an input database.
  • A virtual UsagePoint is created on each PowerElectronicsUnit when applying scenarios, with connectionCategory "SCENARIO_DER". This is done to help the profile service in EWB correlate the unit with a profile in the input database and a capacity. Each added UsagePoint will have a name ("LOADPOINT", "NORMALISED-<PV/EV/BESS>:<profile_id of profile in input database>").
  • Added NetworkService.applyLoadScalingFactor(), which sets the ratedPower of CPI and aggregate UsagePoints to the real part of a LoadScalingFactor as a percentage. This field is used in EWB to determine the scaling factor to apply to the profile from the load database.
  • Added ImpedanceNormalizer to NetworkSimplifier so AcLineSegment with similar PLSI can be normalized to have same PLSI when value differences are within a threshold.
  • Added ratingThreshold and useSpanThresholdRating to CommonImpedanceCollapsor to dictate how current is compared for AcLineSegment collapse.
  • Added emergAmpScaling to config for user to set a scaling factor for emergAmps in LineCode.

Breaking Changes​

    • Refactored EwbDataFilePaths:
    • The EwbDataFilePaths class has been refactored into an interface to enhance flexibility and abstraction.
    • A new class, LocalEwbDataFilePaths, has been introduced to specifically handle the resolution of database paths for the local file system.

  • Switch.ratedCurrent has been converted to a double (used to be an integer). Type safe languages will need to be updated to support floating point arithmatic/syntax.

  • JWTAuthoriser.authorise no longer accepts a permissions claims field, instead it will attempt to retrieve claims from the "permissions" field if it exists in the token, or the "roles" field if the "permissions" field doesn't exist.
  • JWTAuthenticator has a new signature to accept a list of trusted domains rather than a single domain, and a JWTMultiIssuerVerifierBuilder rather than a UrlJwkProvider.
  • Auth0AuthHandler has a new signature and no longer accepts a permissionsField to pass onto JWTAuthoriser.authorise. (See above change to JWTAuthoriser.authorise)
  • AuthRoute.routeFactory has a new signature. Now accepts a list of TrustedIssuer's in place of a urlJwkProvider and issuer.
