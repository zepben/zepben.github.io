Breaking Changes​
  • PostgresLoadDataFetcher now takes a Instant instead of a Date for yearStart.
  • LoadDataFetcher now uses suspend functions.
  • PostgresLoadDataFetcher now suspends on a cache hit instead of blocking for synchronization.
  • Renamed TranslationConfig to ModelConfig.
  • CimToOpenDss now takes a ModelConfig as an argument in translate() rather than a constructor argument.
  • OpenDssTranslator now takes a SolveConfig and a OutputConfig as arguments for translate() rather than constructor arguments.
  • CimToOpenDss now takes an additional boolean flag reportMeterVoltages in translate().
  • Three winding transformers with split phase LV are now modelled correctly.
  • Model startTime can now be configured via ModelConfig. This allows fixed time models to be written out with the correct timestamp.
  • CimToOpenDss::translate now returns a CimToOpenDssResult, which contains both the OpenDSS model and a mapping from CIM terminals to OpenDSS meter names.
  • Removed PostgresLoadDataFetcher.
New Features​
  • Added classes to model an OpenDSS circuit.
  • Modelled OpenDSS commands (e.g. Clear, New, Solve).
  • Added CIM to OpenDSS model translation.
  • Added OpenDSS model writer and OpenDSS to Protobuf translation.
  • Added support for load data fetching.
  • Added support for power electronics (PV, EV, BESS).
  • Improved split phase detection (XYN secondary is now created as 3-winding transformer)
  • Shunt compensators are now translated to capacitors.
  • Feeder load aggregates are now modelled as proxy loads in OpenDSS.
  • Substation transformers are now included in the OpenDSS models.
  • Regulator properties (e.g. tap changer controls) are now translated.
  • Equipment from external feeders are now excluded.
  • Added support for config options in the protobuf Job message (e.g. voltageBases, baseFrequency).
  • Added option to enable phaseVolt for meters in CimToOpenDss.
  • CimToOpenDss::translate now returns a mapping from CIM terminals to OpenDSS meter names along with the model.
Enhancements​
  • None.
Fixes​
  • Fixed issue where system timezone changes the results by shifting them by a number of hours forwards/backwards.
  • Ingress and egress readings are now interpreted in Wh instead of average W over the interval length.
  • Fixed tap settings by dividing the tapStepIncrement by 100%. Tap settings are now only on winding 1.
  • RevBand is now set on RegControls, solving an issue with regulators.
  • Fixed issue where baseVoltage of transformer ends were ignored in translation from CIM to OpenDSS.
  • Fixed incorrect connectivity of loads. Only the phases on the underlying EC/PEC's terminals will be used.
  • Fixed issue where maxIter and maxControlIter are excluded from the OpenDSS script.
  • Set model=7 for all Generators.
Notes​
  • Removed logging of load fetching progress.\n\n---\n

Breaking Changes​

  • Updated the LoadShape message to use a repeated float field rather than a byte array.
  • Handle strings in child class parameters rather than InstantiableElement and fix XfmrCode output
  • Added the new ResultsConfig message to encapsulate different Results Types and used it in the Syf and Job messages.

New Features​

  • Update OutputConfig to work with new ResultsConfig messages.
  • Arbitrary JSON configuration is now passed through the Syf to the executor and the results processor.
  • Added ability to pass a load scaling factor to the CimToOpenDss for load scaling.

Enhancements​

  • Meter names are prefixed with hv/lv and asset category (feeder head, recloser/fuse/loadbreakswitch/etc., tx, calibration)
  • Added endTime property to ModelConfig to specify the end of load time period.
  • Added fixedTime property to ModelConfig to specify single hour load time.
  • Added timezone property to ModelConfig to specify timezone for load time data.
  • Changed IdentifiedObject.elementUid extension to use mRID for equality instead of structure when retrieving objects from the map

Fixes​

  • Fixed bug where energy meters are placed on both sides of an LV switch due to being in a loop. Only the terminal directly facing the distribution transformer will get an energy meter.
  • Explicitly set conn=wye on loads and generators to stop OpenDSS doing its own thing
  • Force vMaxPu to 2 for PV Generators

Notes​

  • Calibration meters are now added into the model regardless of whether their corresponding customer has load.
  • Updated to version 0.2.0 of the hosting-capacity-utils\n\n---\n

Breaking Changes​

  • Looped LV Circuits will no longer have EnergyMeters created for them to avoid EnergyMeters with nonsense results. You will have to infer results from the distribution transformer where looped LV circuits exist.

  • Added support for enabling rudimentary closed loop voltage control through the GeneratorConfig When closedLoopVRegEnabled is true, zone transformers will be configured to provide voltage control based on the voltage at the furthest HV point in the feeder from the zone transformer.

Breaking Changes​

  • From hosting-capacity-utils:
    • Added field intervention: InterventionConfig? = null to WorkPackageConfig and WorkPackageRequest.
    • Added optional field transformerTapSettings to ModelConfig.
    • All timestamp strings being parsed to Kotlin/Java types must include seconds. This is due to the update of kotlinx-datetime to 0.6.1.
    • Added new configuration class SyfConfig to group variables moved out of WorkPackageRequest and WorkPackageConfig.
      • ForecastConfig - Functionally similar to old set up where it holds a list of feeders, years, scenarios and load time.
      • FeederConfigs - Holds a list of individual feeder settings so each feeder can run a hosting capacity study with distinct settings.
    • Added new configuration class LoadTime and LocalLoadTime which holds load specific variables.
      • FixedTime/LocalFixedTime - Fixed time setting for the study, includes an optional map of FixedTimeLoadOverrides.
      • TimePeriod/LocalTimePeriod - Time period setting for the study, includes an optional map of TimePEriodLoadOverrides.
    • Major refactoring of configuration.
      • WorkPackageRequest
        • Moved feeders, years, scenarios, timePeriod and fixedTime into SyfConfig and LocalLoadTime.
      • WorkPackageConfig
        • Moved feeders, years, scenarios, timePeriod and fixedTime into SyfConfig and LocalLoadTime.
      • CommonConfig
        • Moved timePeriod and fixedTime into LoadTime.
      • ModelConfig
        • Moved fixedTimeLoadOverride and timePeriodLoadOverride into LoadTime and LocalLoadTime

New Features​

  • Added field CimToOpenDssResult.conductingEquipmentToLoadUIDs to support load reshaping after running CimToOpenDss.translate().
  • Distribution Transformers will now create RegControl in the model when tap changer control is present.
  • Added new classes to support LvStatCom intervention.
    • InvControl - an InvControl controls all STATCOM pvsystems
    • PvSystem - representation of a pv system
    • XYCurve - a control curve of an InvControl
  • CimToOpenDss.translate() now takes an optional LvStatComInstance. When specified, a PvSystem is created for each StaticVarCompensator in the CIM model with the electrical properties listed in the LvStatComInstance.
  • Distribution Transformers will now create RegControl in the model when tap changer control is present.
  • Supports passing of default load data to be used when no load data is found in the database.
  • From hosting-capacity-utils:
    • Added intervention config to work packages (config and request DTOs). This config is relevant for both the PRRP and the model generator.
  • Adds access to PhaseRebalanceProcessor for intervention.
  • Distribution Transformers will now create RegControl in the model when tap changer control is present.
  • CimToOpenDss.translate() now takes an optional list of BessAllocation. When specified, a Storage, StorageController, and XYCurve are created and added to the model. If the bess's control mode is profile, then a LoadShape object will also be assigned or created.
  • Added new classes to support LvStatCom intervention.
    • Storage - a class that represent an energy storage
    • StorageController - a control for energy storages
  • A list of upstream meter zones is now included in each MeterZoneInfo returned by calculateMeterZoneInfos.
  • Added functionality to take in TimePeriodLoadOverride or FixedTimeLoadOverride to override load profile of specific meter.

Enhancements​

  • Update to utilize new tracing functionality
  • Added support to TestNetworkBuilder for:
    • withClamp - Adds a clamp to the previously added AcLineSegment
    • withCut - Adds a cut to the previously added AcLineSegment
    • connectTo - Connects the previously added item, rather than having to specify it again in connect.
  • Skip creating Loads and Generators for EnergyConsumers and PowerElectronicsConnections with missing phases.

Fixes​

  • From hosting-capacity-utils: Circuit impedance is now scaled by the squared voltage ratio between the primary and secondary windings of inserted virtual transformer.
  • Fix incorrect minTap and maxTap settings for substation transformer. minTap and maxTap should be specified in terms of pu volts rather than ratios. As a result minTap should always be less than maxTap.
    • Tap settings are now always set on the primary winding rather than the secondary
  • Increase rated current of switches to 1000A to avoid unknown switch ratings causing constraints to be identified in the results\
  • SetDirection now correctly sets directions for networks with BusbarSection, Cut and Clamp.
  • The follow fixes were added to Traversal and NetworkTrace:
    • canStopAtStartItem now works for branching traversals.
    • Traversal start items are added to the queue before traversal starts, so that the start items honour the queue type order.
    • Stop conditions on the NetworkTrace now are checked based on a step type, like QueueCondition does, rather than by checking canActionItem.
    • Cut and Clamp are now correctly supported in SetDirection and DirectionCondition.
    • NetworkTrace now handles starting on Cut , Clamp, and AcLineSegment and their terminals in a explicit / sensible way.
    • NetworkTracePathProvider now correctly handles next paths when starting on a Clamp terminal.
  • NetworkTraceStepPathProvider now correctly handles:
    • Next paths when starting on a Clamp terminal.
    • Traversing AcLineSegments with single cuts or clamps.
  • Properly handle irregular traced phases in the OpenDSS bus connections e.g. bus.2.3.1 for traced phases A → B, B → C, C → A.
  • Customers without A/B/C phases are excluded when applying scenarios, preventing problems resulting from missing phases.
  • mRIDs with spaces in them will now be replaced with underscores.

Notes​

  • Updated hosting-capacity-utils version to one that supports the following interventions:
    • Tap optimization
    • Phase rebalancing
    • DVMS
  • evolve-sdk is now a transitive dependency via hosting-capacity-utils.
    • Support up to evolve-sdk-jvm-0.26.0\n\n---\n
