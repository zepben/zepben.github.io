Release History​

VersionReleased
0.1.013 September 2023
Breaking Changes​
  • PostgresLoadDataFetcher now takes a Instant instead of a Date for yearStart.
  • LoadDataFetcher now uses suspend functions.
  • PostgresLoadDataFetcher now suspends on a cache hit instead of blocking for synchronization.
  • Renamed TranslationConfig to ModelConfig.
  • CimToOpenDss now takes a ModelConfig as an argument in translate() rather than a constructor argument.
  • OpenDssTranslator now takes a SolveConfig and a OutputConfig as arguments for translate() rather than constructor arguments.
  • CimToOpenDss now takes an additional boolean flag reportMeterVoltages in translate().
  • Three winding transformers with split phase LV are now modelled correctly.
  • Model startTime can now be configured via ModelConfig. This allows fixed time models to be written out with the correct timestamp.
  • CimToOpenDss::translate now returns a CimToOpenDssResult, which contains both the OpenDSS model and a mapping from CIM terminals to OpenDSS meter names.
  • Removed PostgresLoadDataFetcher.
New Features​
  • Added classes to model an OpenDSS circuit.
  • Modelled OpenDSS commands (e.g. Clear, New, Solve).
  • Added CIM to OpenDSS model translation.
  • Added OpenDSS model writer and OpenDSS to Protobuf translation.
  • Added support for load data fetching.
  • Added support for power electronics (PV, EV, BESS).
  • Improved split phase detection (XYN secondary is now created as 3-winding transformer)
  • Shunt compensators are now translated to capacitors.
  • Feeder load aggregates are now modelled as proxy loads in OpenDSS.
  • Substation transformers are now included in the OpenDSS models.
  • Regulator properties (e.g. tap changer controls) are now translated.
  • Equipment from external feeders are now excluded.
  • Added support for config options in the protobuf Job message (e.g. voltageBases, baseFrequency).
  • Added option to enable phaseVolt for meters in CimToOpenDss.
  • CimToOpenDss::translate now returns a mapping from CIM terminals to OpenDSS meter names along with the model.
Enhancements​
  • None.
Fixes​
  • Fixed issue where system timezone changes the results by shifting them by a number of hours forwards/backwards.
  • Ingress and egress readings are now interpreted in Wh instead of average W over the interval length.
  • Fixed tap settings by dividing the tapStepIncrement by 100%. Tap settings are now only on winding 1.
  • RevBand is now set on RegControls, solving an issue with regulators.
  • Fixed issue where baseVoltage of transformer ends were ignored in translation from CIM to OpenDSS.
  • Fixed incorrect connectivity of loads. Only the phases on the underlying EC/PEC's terminals will be used.
  • Fixed issue where maxIter and maxControlIter are excluded from the OpenDSS script.
  • Set model=7 for all Generators.
Notes​
  • Removed logging of load fetching progress.\n\n---\n
