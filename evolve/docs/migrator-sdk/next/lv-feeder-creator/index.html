Lv Feeder Creator

The LvFeederCreator is used to create LvFeeders on a given NetworkService. An LvFeeder is an EquipmentContainer that represents a branch of LV network starting at a distribution substation and continuing until the end of the LV network. LvFeeders are associated with Feeders and typically a Feeder will have many LvFeeders, with one per distribution substation. AssignToLvFeeders in the EWB SDK should be run after AssignToFeeders to associate equipment with the generated LvFeeders.

The distribution substation will belong to both the Feeder associated with the LvFeeder and the newly created LvFeeder, with the normalHeadTerminal of the LvFeeder pointing to the LV side Terminal of the distribution substation.

This processor should be run on any NetworkService that contains LV network, and will create an LvFeeder on all terminals that bridge HV and LV equipment.

*** Note: Terminals that are already the head of an LvFeeder, or LV terminals that originate in a Substation will be skipped.

Prior to running the LvFeederCreator your NetworkService must meet the following conditions:

  • The base voltage of each equipment must be correctly assigned. Use the VoltageProcessor to ensure all equipment has voltages.
  • The connectivity of conducting equipment must be final.
  • All HV Feeders you wish to add from the source data have been added. The LvFeeders can be associated with upstream Feeders through AssignToLvFeeders
  • All LvFeeders you wish to include from the source data have been added with their normalHeadTerminal set to the LV side of a HV -> LV bridge. This will ensure duplicate LvFeeders are not created unnecessarily.
  • All equipment inside a substation has been added to the Substation

Usage

Create an LvFeederCreator and process a NetworkService. This will yield LvFeeders for any distribution substations in networkService with an mRID of "[distribution-sub-lv-terminal-mRID]-lvf"

  • Kotlin
val lvFeederCreator = LvFeederCreator()

lvFeederCreator.process(networkService)

Create an LvFeederCreator with an mRIDSelector that uses the normalHeadTerminals associated ConductingEquipment's mRID for the LvFeeder mRIDs.

  • Kotlin
val lvFeederCreator = LvFeederCreator { "${it.conductingEquipment.mRID}-lvf" }

lvFeederCreator.process(networkService)
