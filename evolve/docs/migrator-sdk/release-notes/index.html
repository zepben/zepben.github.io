[0.25.0]â€‹

Breaking Changesâ€‹

  • Updated abstract class TapSettingsProcessor. readTaps() now must return a Map<String, TxTapRecord> and implementations must provide a sourceName for logging purposes.
  • Updated abstract class VoltageRegulatorSettingsProcessor to be concrete and make use of the logic for processing CPPAL regulator settings.
  • You can no longer register a StepActionWithContextValue with Traversal.ifStopping or Traversal.ifNotStopping, you must use Traversal.addStepCondition instead.
  • Added a new debugLogging and name parameters to the constructor of the following traces. The helper functions in Tracing also have these parameters, which defaults to null and networkTrace, meaning anyone using these wrappers will be unaffected by the change:
    • AssignToFeeders
    • AssignToLvFeeders
    • ClearDirection
    • FindSwerEquipment
    • PhaseInferrer
    • RemovePhases
    • SetDirection
    • SetPhases
  • NetworkStateOperators has a new abstract description. If you are creating custom operators you will need to add it.
  • A new phase allocation algorithm has been added to the PhaseConsistencyProcessor (and XyPhaseProcessor):
    • The new chooseReplacements algorithm takes the PhaseIsland and a list of valid ReplacementPhases, returning the ReplacementPhases to use for the island. It is used to determine the replacement set of phases that applies to the entire island and caters for all phase types (HV1, SWER, LV2 and LV1). There is a default for this, so you may not need to update the code, but it might (or probably will) change the phases selected. This will cause an issue if you have specified a function for the previous phaseAllocator.
    • Due to adding a new algorithm, and to avoid confusion, the previous phaseAllocator has been renamed choosePhases. It is used to determine which LV1 or SWER phase to use when there are multiple options available in the given replacement set. This will only impact you if are using named parameters.
  • Removed public access to XyPhaseProcessor.replaceFromKnownStart and Iterable.toLinkedHashSet which aren't being used externally.
  • XYPhaseProcessor has been moved to com.zepben.migrator.processors.phasing.xy, you will need to update your imports. This has no impact if you are only using the processor via PhaseConsistencyProcessor.makeConsistent.

New Featuresâ€‹

  • You can now pass a logger to all Tracing methods and TestNetworkBuilder.build to enable debug logging for the traces it runs. The debug logging will include the results of all queue and stop condition checks, and each item that is stepped on.

Enhancementsâ€‹

  • NetworkTrace now supports starting from a known NetworkTraceStep.Path. This allows you to force a trace to start in a particular direction, or to continue a follow-up trace from a detected stop point.
  • Support disabling certificate verification in SDK connection helpers
  • Phase consistency processing has been updated to support cuts and clamps.

Fixesâ€‹

  • QueryNetworkStateService.reportBatchStatus now correctly sends the Empty response.
  • Phases are now correctly assigned to the LV side of an LV2 transformer that is in parallel with a previously energised LV1 transformer.

    • Stopped assigning N to service equipment with no known phases.
    • Allowed sub-transmission transformers to detect their types.

  • NetworkTrace/Traversal now correctly respects canStopOStart when providing multiple start items.

  • Updated to latest JVM SDK - see SDK release notes for versions 0.15.0 and 0.16.0 for changes.
  • FeederTerminalTracer:
    • findFeederHeadTerminal now detects the sub-transmission side by looking for a PowerTransfofmer that is assigned to a Substation rather than one with a function of powerTransformer. This means you must ensure substation transformers are correctly assigned a Substation.
    • findFeederSourceTerminal now only takes a Feeder to create a source terminal for. We also explicitly require that a Feeder has a normalHeadTerminal set.
  • EnergySourceProcessor:
    • createSources now detects substation transformers by an explicit link to the Substation rather than applying a source to all transformers with a function of powerTransformer. This means you must ensure substation transformers are correctly assigned a Substation.
    • createSource now takes either a Feeder or a PowerTransformer, and no longer takes feeder start nodes. When providing a Feeder, we explicitly require the normalHeadTerminal be populated.
    • createSource and createSources are now accepting a sourceImpedanceFetcher instead of the impedance map. This callback should fetch and return the desired SourceImpedance for the given ConductingEquipment, which will be either the head equipment for a Feeder or the zone PowerTransformer if available.
  • The getTransformerPhaseType callback of the PhaseConsistencyProcessor now provides the PowerTransformer, not just its mRID.

New Featuresâ€‹

  • Added NetworkSimplifier, which simplifies the network to prepare it for direct translation to a bus-branch model.
    • This reduces the set of network elements to a few simple types (lines, transformers, energy sources, PECs, and consumers).
    • Switches may be included or excluded using the keepSwitches parameter (defaults to false).
    • Excluded elements are treated as zero-impedance and collapsed into connectivity nodes, unless the element is an open switch, in which case it is removed entirely.
    • Strings of line segments with identical impedance, phasing, and cable info are collapsed into a larger line.
    • Lines with negligible impedance are usually collapsed, unless they connect a shunt compensator.
    • To make translation to certain bus-branch models (e.g. OpenDSS) simpler, it is ensured that each terminal is connected to a connectivity node.
    • If a feeder head terminal is on equipment that is excluded (e.g. a breaker), the NetworkSimplifier attempts to relocate it to a terminal connected to the connectivity node it collapsed into.
  • Added callbacks to the PhaseConsistencyProcessor allowing you to inject debug code without having to release the library. It is not expected that any production code will be required in these callbacks.
  • Created a VoltageConverter to change between common L-N and L-L voltages.
  • Added TerminalTracer which should be moved to the SDK when reworking all the traces.
  • Added option to the PhaseConsistencyProcessor and TransformerPhaseProcessor to force the LV network below a 3-phase transformer to ignore the phasing of the backbone and replace it with 3-phase.

v0.1.0â€‹

Initial Release

Breaking Changesâ€‹

  • Moved EnergySourceProcessor from package com.zepben.migrator.processors to com.zepben.migrator.processors.sources.
  • You now need to provide a SourceIssues instance to EnergySourceProcessor.
  • Fix the ordering for the processing of TerminalReplacement instances from breadth first to depth first in the IncompatiblePhaseFixer. This prevents the phase fixing code from getting stuck in ABC to ABCN loops.
  • Renamed FeederDirection.has to FeederDirection.contains, which can be used via its operator version in. e.g. BOTH.has(DOWNSTREAM) can be replaced with BOTH.contains(DOWNSTREAM) or DOWNSTREAM in BOTH

New Featuresâ€‹

  • Added LvFeederCreator, which adds missing LV feeders to the secondary terminals of each distribution transformer.
  • Added the following constants to AcLineSegmentDefaults for use with things such as internal conductors that are not real artefacts, or busbars or other zero length cables/lines that have no provided impedance, type or rating details:
ConstantDescription
ZEPBEN_ZERO_IMP_PLSIA zero-impedance PerLengthLineImpedance used to give no impedance to a line, while also indicating that no impedance data was provided.
ZEPBEN_UNRATED_WIRE_INFOAn OverheadWireInfo that has such a high rating it is essentially unrated, while also indicating that no cable type or rating information was provided.
  • Added VoltageProcessor - a helper processor that wraps the VoltageLevelAssigner and VoltageLevelVoltageAssigner
  • Added support for current transformers and power transformers with the following classes in com.zepben.evolve.cim.*:
    • In com.zepben.evolve.cim.iec61968.infiec61968.infassetinfo:
      • CurrentTransformerInfo: Properties of current transformer asset.
      • PotentialTransformerInfo: Properties of potential transformer asset.
    • In com.zepben.evolve.cim.iec61968.infiec61968.infcommon:
      • Ratio: Fraction specified explicitly with a numerator and denominator, which can be used to calculate the quotient.
    • In com.zepben.evolve.cim.iec61970.base.auxiliaryequipment:
      • CurrentTransformer: Instrument transformer used to measure electrical qualities of the circuit that is being protected and/or monitored.
      • PotentialTransformer: Instrument transformer (also known as Voltage Transformer) used to measure electrical qualities of the circuit that is being protected and/or monitored.
      • PotentialTransformerKind: The construction kind of the potential transformer. (Enum)
      • Sensor: This class describes devices that transform a measured quantity into signals that can be presented at displays, used in control or be recorded.
  • Added PowerTransformer().getEnd(Terminal), which gets a PowerTransformerEnd by the Terminal it's connected to.
  • Added the following functions to ConnectedEquipmentTrace for creating traces that work on ConductingEquipment, and ignore phase connectivity, instead considering things to be connected if they share a ConnectivityNode:
    • newNormalDownstreamEquipmentTrace: Creates a trace that traverses in the downstream direction using the normal state of the network.
    • newNormalUpstreamEquipmentTrace: Creates a trace that traverses in the upstream direction using the normal state of the network.
    • newCurrentDownstreamEquipmentTrace: Creates a trace that traverses in the downstream direction using the current state of the network.
    • newCurrentUpstreamEquipmentTrace: Creates a trace that traverses in the upstream direction using the current state of the network.
  • Added support for protection equipment with the following classes, enums, and fields:
    • SwitchInfo: Switch datasheet information.
    • ProtectionEquipment: An electrical device designed to respond to input conditions in a prescribed manner and after specified conditions are met to cause contact operation or similar abrupt change in associated electric control circuits, or simply to display the detected condition.
    • CurrentRelay: A device that checks current flow values in any direction or designated direction.
    • CurrentRelayInfo: Current relay datasheet information.
    • RecloseSequence: A reclose sequence (open and close) is defined for each possible reclosure of a breaker.
    • ProtectionKind: The kind of protection being provided by this protection equipment.
    • ProtectedSwitch::breakingCapacity: The maximum fault current in amps a breaking device can break safely under prescribed conditions of use.
    • Switch::ratedCurrent: The maximum continuous current carrying capacity in amps governed by the device material and construction. The attribute shall be a positive value.
    • Breaker::inTransitTime: The transition time from open to close in seconds.
  • Added DTOs of the following CIM classes to support custom default configurations:
    • PerLengthSequenceImpedance
    • WireInfo - translates to either CableInfo or OverheadWireInfo depending on the isOverhead flag.
    • PowerTransformerInfo, along with every directly and transitively referenced class (PowerTransformerInfo, TransformerEndInfo, etc.).
  • AcLineSegmentDefaults now supports custom defaults for wireInfo and perLengthSequenceImpedance, configurable via the new constructor parameters defaultWireInfo and defaultPerLengthSequenceImpedance respectively.
  • PowerTransformerDefaults now supports a custom default for assetInfo, configurable via the new constructor parameter defaultPowerTransformerInfo.

Enhancementsâ€‹

  • Updated to the latest super pom.
  • Energy sources created on feeder heads or substation transformers are now added to the Substation.
  • Added support to the TransformerPhaseProcessor for HV1 to LV2 transformers.
  • Added support for split 3-phase backbones.
  • Updated the phase consistency processing to prevent incorrect phases in the backbone of a feeder from severely breaking the feeder phasing by checking downstream transformer phases for compatibility before spreading them. If they are not compatible with the transformers, repair the upstream path instead.
  • Added FeederDirection.not operator function.

Fixesâ€‹

  • Removed excess warnings from the TransformerPhaseProcessor about conflicting SWER voltages and phases when the transformer phasing is unknown.
  • Prevented the IncompatiblePhaseFixer from removing valid phases when restoring the upstream path to a transformer.
  • NetworkService.remove for LvFeeder function fixed (previously added instead).
  • Fixed bug where limited connected traces with maximumSteps = 1 could include equipment 2 steps away when using a direction.
  • The evolve-sdk now correctly assign AuxiliaryEquipment and ProtectionEquipment to Feeders and LvFeeders.
