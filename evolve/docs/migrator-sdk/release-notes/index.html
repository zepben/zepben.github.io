NOTE: This library is not yet stable, and breaking changes should be expected until a 1.0.0 release.

Breaking Changes
  • Updated SDK with breaking changes.
  • Conversion of StdTwoWindingTransformer and StdThreeWindingTransformer now stores the type name in PowerTransformerInfo.description rather than TransformerEndInfo.description
  • Reorganised the processors package. This caused the packages of a lot of the classes to change. Please re-import any affected classes from their new locations.
  • Merged in the processing of energy sources from the GIS extractor. Energy sources will now be created only on feeders with no EHV, and on power transformers within EHV.
  • The EnergySource that is added to a PowerTransformer by the EnergySourceCreator will now be added to the terminal with the lowest HV voltage, with the most phases.
New Features
  • Added PhaseConsistencyProcessor, which can be used to fix inconsistent phases in a NetworkService. This includes the following sub-processors:
    • DroppedPhaseLoopFixer
    • IncompatiblePhaseFixer
    • PhaseExtender
    • TransformerPhaseProcessor
    • XyPhaseProcessor
  • Added CroProcessorPoF, which can be used to extract OperationalRestriction instances from PowerOn Fusion/Advantage.
  • Added DefaultValueProcessor, which can be used to run a set of DefaultValueAssigner instances.
  • Added DuplicatePathConductorRemover, which can be used to remove duplicate conductors which share the same equipment on both ends.
  • Added BypassedSwitchFixer, which removes any conductors which connect to multiple terminals of a single switch.
Enhancements
  • Updated IssueTrackerGroup to support finding trackers from delegated classes.
Fixes
  • Subclasses of IssueTrackerGroup now find trackers from their superclasses.
Notes
  • None.

v0.8.0

Breaking Changes
  • Reorganised packages between com.zepben.migrator and com.zepben.verifier:
  • Moved com.zepben.migrator.datachecker to com.zepben.verifier.datachecker.
  • Moved com.zepben.verifier.processors to com.zepben.migrator.processors.
    • Excluded VoltageLevel from this move, instead moving it to com.zepben.verifier.
    • Added assigners.EnergySourceProcessor to this package.
  • Moved com.zepben.migrator.Services to com.zepben.verifier.Services.
  • Updated network verifier library with breaking changes.
New Features
  • Added the following classes that are common to RDF importers:
    • RdfElement, RdfAttribute, RdfMapper, and RdfMappingProcessor, under the rdf package.
  • Added TestExtensions for the issues package to allow simple testing of summary line generation.

  • None.
Fixes
  • None.

v0.7.0

Breaking Changes
  • Added assignment of default WireInfo based on voltage to AcLineSegmentDefaults.
New Features
  • Phases are now set through transformers allowing for phase changes. Only valid phase mappings are currently supported. If a valid mapping has been missed, please contact Zepben.
  • Added an AcLineSegment filter for the lines getting defaults applied using AcLineSegmentDefaults.
  • Added an PowerTransformer filter for the txs getting defaults applied using PowerTransformerDefaults.
New Features
  • Added IssuesLog.
Enhancements
  • None.
Fixes
  • SetDirection now correctly handles Switch instances without phases.
  • SetPhases no longer removes phases when there is an LV loop with mixed single and 3-phase.
Notes
  • None.
