Metrics for Issues

Motivation​

Issue trackers are useful for collecting and displaying problems with source data, but their usefulness is limited if these issues are not saved somewhere after the run of an ingestor. This is why we added the ability for issue trackers to add metrics to the metrics database that correspond to each issue.

Network Container Metrics for Issues​

Network container metrics store quantitative metrics for network containers, e.g. feeders and substations. By connecting an IssueTracker to a NetworkMetrics object and supplying a metricName, a metric with that name will be incremented each time the track() method is called for that IssueTracker. By default, this metric will only be incremented for the TotalNetworkContainer, but you may supply additional NetworkContainers in the networkContainers parameter.

  • Kotlin
import com.zepben.evolve.cim.iec61970.base.core.Feeder
import com.zepben.evolve.metrics.NetworkMetrics
import com.zepben.evolve.metrics.TotalNetworkContainer
import com.zepben.evolve.metrics.toNetworkContainer
import com.zepben.migrator.issues.IssueTracker
import org.slf4j.Logger
import org.slf4j.LoggerFactory

val logger: Logger = LoggerFactory.getLogger("Example")

fun main() {
    val networkMetrics = NetworkMetrics()
    val exampleIssue = IssueTracker(logger, networkMetrics = networkMetrics, metricName = "Example Issue") { "issue was tracked $it times" }
    val feeder = Feeder("FDR123").apply { name = "Feeder 123" }

    exampleIssue.track("This message won't be saved to the network metrics")
    exampleIssue.track("Neither will this one", networkContainers = listOf(feeder.toNetworkContainer()))

    println("Number of times example issue was tracked in total: ${networkMetrics[TotalNetworkContainer]["Example Issue"]}")
    println("Number of times example issue was tracked for the feeder: ${networkMetrics[feeder.toNetworkContainer()]["Example Issue"]}")
}

For issue tracker groups, all issue trackers within the group can be assigned a common NetworkMetrics and a default name with a single call to setNetworkMetrics(). The property name for each issue tracker within the group will be used as their default name:

  • Kotlin
import com.zepben.evolve.cim.iec61970.base.core.Feeder
import com.zepben.evolve.metrics.NetworkMetrics
import com.zepben.evolve.metrics.TotalNetworkContainer
import com.zepben.evolve.metrics.toNetworkContainer
import com.zepben.migrator.issues.IssueTracker
import com.zepben.migrator.issues.IssueTrackerGroup
import org.slf4j.Logger
import org.slf4j.LoggerFactory

val logger: Logger = LoggerFactory.getLogger("Example")

fun main() {
    val networkMetrics = NetworkMetrics()
    val exampleIssueTrackerGroup = object : IssueTrackerGroup() {
        val exampleIssue = IssueTracker(logger) { "issue was tracked $it times" }
    }.apply { setNetworkMetrics(networkMetrics) }
    val feeder = Feeder("FDR123").apply { name = "Feeder 123" }

    exampleIssueTrackerGroup.exampleIssue.track("This message won't be saved to the network metrics")
    exampleIssueTrackerGroup.exampleIssue.track("Neither will this one", networkContainers = listOf(feeder.toNetworkContainer()))

    println("Number of times example issue was tracked in total: ${networkMetrics[TotalNetworkContainer]["exampleIssue"]}")
    println("Number of times example issue was tracked for the feeder: ${networkMetrics[feeder.toNetworkContainer()]["exampleIssue"]}")
}
