Config File

Configuration for the Load Data Ingestor is specified via a single JSON file.

Sample Configuration​

Not all available config fields are featured in these sample configurations. See Configuration Specification for more details.

Ingest and Aggregate Meter Readings​

{
  "io": {
    "loadDatabase": {
      "host": "host",
      "port": "port",
      "dbName": "db",
      "schema": "schema",
      "username": "user",
      "password": "password"
    },
    "inputStorage": {
      "type": "azure",
      "storageAccountName": "storageAccountName",
      "storageContainerName": "storageContainerName"
    }
  },
  "actions": [
    {
      "type": "IngestMeterReadings",
      "company": "Company-Name",
      "inputPathPrefix": "prefix/to/match",
      "inputPathPattern": "<regex-to-match-after-prefix>"
    },
    {
      "type": "AggregateMeterReadingsToCpi"
    },
    {
      "type": "AggregateMeterReadingsToAup",
      "useHistoricMapping": false
    },
    {
      "type": "CompressMeterReadings"
    },
    {
      "type": "CompressCpiAggregations"
    },
    {
      "type": "CompressUsagePointAggregations"
    }
  ]
}

Configuration Specification​

Top Level​

FieldDescriptionTypeDefault
ioConfigures connections to input and output data storage.IO
trackersConfigures in-memory data that persists throughout the execution of the run and are used as the input of some actions.Trackerssee Trackers
actionsA list of actions that will be run in-sequence. If any action fails to execute, the run ends early.Array of Actions
forceDeleteWhether to process files even after they were successfully processed in a previous run. Defaults to false.Booleanfalse

IO​

FieldDescriptionTypeDefault
loadDatabaseConfigures connection to the load database to output to. Not strictly required for ingestion actions, but nothing will be written to if this is null.null or Load Database
inputStorageConfigures connection to the blob storage where the input data is stored. Required if any of the actions are ingestion.null or Input Storage
concurrencyLimitMaximum number of threads to work on reading/writing/processing data in parallel.Positive Integer2 * number of logical cores

Load Database​

FieldDescriptionTypeDefault
hostHostname of the database.String
portPort of the database.Integer in [0, 65535]
dbNameName of the database.String
schemaSchema where the table sits (default to public)String"public"
usernameUsernameString
passwordPasswordString

Input Storage​

One of the following:

Azure​

FieldDescriptionTypeDefault
typePut as "azure"String "azure"
storageAccountNameStorage Account Name.String
storageContainerNameStorage Container Name.String
localStoreDirectory where files are downloaded to.StringTemporary directory

S3​

FieldDescriptionTypeDefault
typePut as "s3"String "s3"
regionRegion where the s3 bucket sits.null or Stringnull
bucketNameName of the s3 bucket.String
localStoreDirectory where files are downloaded to.StringTemporary directory

Local​

FieldDescriptionTypeDefault
typePut as "local"String "local"
newFileStoreDirectory where the input files are.StringCurrent working directory

Trackers​

FieldDescriptionTypeDefault
cpiAggregationThresholdIf the number of CPIs with new readings reaches this number, all CPIs are aggregated in the AggregateMeterReadingsToCpi action, rather than only the updated ones.Nonnegative Integer10000

Action​

All actions support the following field:

FieldDescriptionTypeDefault
descriptionCustom configuration to use for the action, to be displayed in logs.StringSensible description depending on type and config of action

Each action is one of the following:

Drop CPI Aggregations​

FieldDescriptionTypeDefault
typePut as "DropCpiAggregations"String "DropCpiAggregations"
dateRangeIf specified, only CPI aggregations within this date range are dropped.null or Date Rangenull

Drop Usage Point Aggregations​

FieldDescriptionTypeDefault
typePut as "DropUsagePointAggregations"String "DropUsagePointAggregations"
dateRangeIf specified, only usage point aggregations within this date range are dropped.null or Date Rangenull

Ingest Meter Readings​

FieldDescriptionTypeDefault
typePut as "IngestMeterReadings"String "IngestMeterReadings"
companyCompany name, used to determine format of input data.One of "UE", "ENDEAVOUR", "PCOR", "EE"
inputPathPrefixIf specified, only blobs at paths starting with this prefix are read.null or Stringnull
inputPathPatternPattern to match paths with, after removing the matching inputPathPrefix if it is specified.Regex String".*"
ignoreSuccessFilesWhether to ignore the .success files that are generated after a file is processed successfully.Booleanfalse
detailedExtractorLoggingWhether to log each individual issue that happens during ingestion.Booleanfalse
batchSizeMaximum size of each batch written to the load database.Positive Integer10000
batchCacheLimitMaximum size of the cache holding meter readings being batchedPositive Integer250000

Ingest PQV​

FieldDescriptionTypeDefault
typePut as "IngestPqv"String "IngestPqv"
companyCompany name, used to determine format of input data.One of "UE", "ENDEAVOUR", "PCOR", "EE"
inputPathPrefixIf specified, only blobs at paths starting with this prefix are read.null or Stringnull
inputPathPatternPattern to match paths with, after removing the matching inputPathPrefix if it is specified.Regex String".*"
ignoreSuccessFilesWhether to ignore the .success files that are generated after a file is processed successfully.Booleanfalse
detailedExtractorLoggingWhether to log each individual issue that happens during ingestion.Booleanfalse
batchSizeMaximum size of each batch written to the load database.Positive Integer10000

Aggregate Meter Readings to CPI​

FieldDescriptionTypeDefault
typePut as "AggregateMeterReadingsToCpi"String "AggregateMeterReadingsToCpi"
aggregateAllIf true, all CPIs will be aggregated regardless of the load data ingested in previous IngestMeterReadings actions.Booleanfalse
forceDateRangeIf specified, Only dates within this date range will be aggregated. Must be null if aggregateAll is false.null or Date Rangenull

Aggregate Meter Readings to AUP​

FieldDescriptionTypeDefault
typePut as "AggregateMeterReadingsToAup"String "AggregateMeterReadingsToAup"
useHistoricMappingIf false, the most recent mappings are considered rather than the ones relevant for each timestep being aggregated.Booleantrue
forceDateRangeIf specified, Only dates within this date range will be aggregated.null or Date Rangenull
