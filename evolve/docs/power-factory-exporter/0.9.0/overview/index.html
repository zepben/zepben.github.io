power-factory-exporter

Power factory exporter library - Maps network data from Cim to power factory compatible DGS format.

Starting 🚀​

These instructions will allow you to get a copy of the project running on your local machine for development and testing purposes and/or running a docker container as well.

Maven​

Add to your pom.xml:

<dependency>
    <groupId>com.zepben.powerfactory</groupId>
    <artifactId>exporter</artifactId>
    <version>0.7.0</version>
</dependency>

Usage​

For a basic example of using the power factory exporter refer to the demonstration code in file com.zepben.powerfactory.exporter.demo.Demo.kt

    val logger: Logger = LoggerFactory.getLogger("main")

    GrpcChannelFactory.create(ConnectionConfig(demoConfig.ewb.host, demoConfig.ewb.grpcPort, demoConfig.ewb.secureConnection)).use { channel ->
        val dgsGenerator = DgsGenerator(DgsGenerator.Config(), XlsxDgsWriter()) { NetworkConsumerClient(channel) }

        when (val result = dgsGenerator.generate(demoConfig.generationConfig)) {
            is DgsGenerationResult.Success<XSSFWorkbook> -> {
                writeFile(result.dgs, "output/model.xlsx")
            }

            is DgsGenerationResult.Failure<*> -> {
                logger.error("Failed to create model with the following errors:")
                result.errors.forEach { (m, t) -> logger.error(m, t) }
            }
        }
    }

Documentation 📦​

Workflow Overview​

  1. User Request Power Factory Model by sending Model Config to Power Factory Model Builder
  2. Model Config sent to Power Factory Exporter
    This library starts here
  3. Request Network Model from Energy Work Bench
  4. Receive Network Model from Energy Work Bench
  5. Return DGS to Power Factory Model Builder
    This library ends here
  6. Sent DGS to Power Factory Template Service
  7. Return DGS to Power Factory Model Builder
  8. Return DGS to EWB App Server

Library Concept​

Given Meta data and a list of equipment containers, this library will map CIM data of equipments belonging to the list of equipment containers to Power Factory Elements.

List of Available Processors​

DgsService post mapping can be put through a list of processors. These processors can improves the clarity of geographic diagrams or help guide the auto draw function of power factory to produce better schematics diagrams. Some of these processors will help with load flow study set up or timescale database linkage.

ProcessorsFunctionality
AdditionalVariablesProcessorProcessor that adds mapping for additional variables based on request.
DatabaseProcessorProcessor create characteristic object to line up ElmLod and timescale database entries
DefaultLoadProcessorProcessor that assign loads to ElmLod base on the instruction or data provided.
ElementGapperProcessorProcessor attempt to put gapping distance between ElmTerm and the Object inbetween it, so it looks presentable and can be toggled in power factory.
ElmCoupProcessorProcessor assigning the correct ElmCoup usage so correct custom icon shows up in the model.
ElmFeederColourProcessorProcessor assigning feeder colour base on an input file so network of same feeder has the same colour.
ElmNetNameProcessorProcessor renaming LV Elmnet for clarity and organization. Part of the aesthetic clean-up.
FinalCleanUpProcessorProcessor that does a round of final clean up of the model to remove potential error and cluster.
LoadSiteProcessorProcessor creating sites and putting Elmlod in them, so they show up as a circle on the main map.
LvConsolidationProcessorProcessor consolidating LV grid into their respective HV grid.
LvElmTermProcessorProcessor that normalize Elmterms downstream of distribution transformers.
NERProcessorProcessor that creates an ElmGndswt and place it downstream of ElmSind.
OverlappingNodesAndEquipmentProcessorProcessor to fix overlapping nodes and equipments that are not directly connected.
RecloserElmFeederProcessorProcessor to assign a functioning feeder head terminal to feeders situated on a breaker in the recloser loop
RecloserProcessorThis is the second half of the recloser site processor, first part is in the importer.
RemoveGpsSiteCoordsProcessorProcessor removing GPS co-ord from equipment in a substation site or energy consumer site so it doesn't clutter the main geographic map.
RemoveHangingElmCoupProcessorProcessor removing Elmcoup (switch) that is only connected on one side. This just remove noise from the substation.
StarPointProcessorProcessor assigning resistance of internal grounding value for distribution transformers.
SubstationEdgeElmTermSortingProcessorProcessor that sort all Elmterm connected to equipments in substations into the substation for diagram clarity.
SubtransProcessorProcessor that consolidate Subtrans circuits into loops and disable non-BSP energy sources.
TransformerProcessorProcessor fix up distribution transformer sites and isolation transformers.
TriSwitchProcessorProcessor that moves tri switch set up from their original point to between their flanking lines.
VariantProcessorProcessor to sort objects created in hosting capacity scenarios into variant stages (IntSstage).

AdditionalVariablesProcessor​

  1. Assign more information label to Open Switches.
  2. Assign more information label to Sites.

Return to Full List of Processors

DatabaseProcessor​

Required: databaseConfig and nmiDatabaseList are set in the config.

  1. A SetDatabase object is created and added to the DgsService base on details provided in the config file.
  2. A ChaTime and ChaRef object is created for each EnergyConsumer if its NMI matches an entry in the provided nmiDatabaseList.

Return to Full List of Processors

DefaultLoadProcessor​

Optional: feederMax and hvcMaxDemand set in the config.

  1. If feederMax and hvcMaxDemand are set in config, load value will be distributed to each ElmLod base on placement. Subtrans Model: An aggregated load is created at the feeder head and provided value is assigned to the aggregated load.
    HV Model: Provided value is split into downstream aggregated load placed immediately downstream of DTX base on DTX size, and HV customer is assigned value base on hvcMaxDemand.
    LV Model: Value that would've been assigned to the DTX aggregated load is instead evenly split up and assigned to all downstream ElmLod.

  2. When feederMax and hvcMaxDemand are not provided, a default load value is assigned to each ElmLod.

Return to Full List of Processors

ElementGapperProcessor​

Connectivity nodes in CIM does not carry location data, thus their placement must be determined by the conducting equipment attached to it. Unfortunately, this will result in overlapping elements in the power factory model. The ElementGapperProcessor will go through objects in the geographic diagram and create proper spacing between these objects to improve model clarity and allow user interaction.

Return to Full List of Processors

ElmCoupProcessor​

The processor goes through each ElmCoup in the DgsService and assign the correct elmCoup.aUsage value base on elmCoup.chrName value.

Return to Full List of Processors

ElmFeederColourProcessor​

The processor goes through each ElmFeeder in the DgsService and assign the correct elmFeeder.iColor value base on the file provided by feederColourPath.

Return to Full List of Processors

ElmNetNameProcessor​

The processor goes through each ElmNet in the DgsService and clean up all DTX names.

Return to Full List of Processors

FinalCleanUpProcessor​

The processor does a final pass on the model to clean up potential errors and reduce cluster.

  1. Remove all energy source with no source impedance
  2. Final pass to ensure elements are sorted into the correct folder
  3. Remove Current transformers that were attached to elements removed during clean-up/post process
  4. Remove unused StaCubic and ElmTerm from grounding switches and assign correct objBus to staCubic that is in use
  5. Remove all Auxiliary equipments from zone sub because they are just clutter
  6. Remove any Stacubic (Terminals) that are not being used by any equipment
  7. Remove any equipment that is not connected to anything
  8. Remove and Elmterm(Connectivity node) that has nothing attached to it
  9. Disable all isolation transformer and auxiliary transformer in substations

Return to Full List of Processors

LoadSiteProcessor​

The processor goes through each ElmLod in the DgsService and create an ElmSite and move it out of the geographic diagram if the ElmLod doesn't belong in an ElmSubstat or ElmTrfstat.

Return to Full List of Processors

LvConsolidationProcessor​

The processor goes through each LvFeeder in the NetworkService and assign their upstream feeder to the corresponding ElmNet.foldId. This will consolidate all LV grid into the HV grid to lower clustering. This processor should be disabled if user want a model with LV section in their own toggleable grid.

Return to Full List of Processors

LvElmTermProcessor​

The processor goes through all DTX in the NetworkService and standardize all downstream ElmTerm ukNom and unkNom value for load flow studies.

Return to Full List of Processors

NERProcessor​

Processor creates an ElmGndswt and place it downstream of ElmSind.

Return to Full List of Processors

OverlappingNodesAndEquipmentProcessor​

The processor goes through all ElmTerm and find ServiceElement with exact identical gpsLat and gpsLon and separates them.

  1. Currently only support movement of ElmCoup

Return to Full List of Processors

RecloserElmFeederProcessor​

The processor goes through each ElmFeeder to check if its head terminal needs to be moved to an AcLineSegment downstream, because the breaker is in a recloser loop.

Return to Full List of Processors

RecloserProcessor​

The processor goes through all reclosers and voltage regulators in the dgsService to rearrange/create/remove components for geographic diagram clarity.

Return to Full List of Processors

RemoveGpsSiteCoordsProcessor​

The processor goes through all ElmTrfstat and ElmSubstat in the DgsService and remove all gpsLon and gpsLat value from ServiceElement belonging to these equipment containers to improve geographic diagram clarity.

Return to Full List of Processors

RemoveHangingElmCoupProcessor​

The processor goes through all elmCoup and relFuse in the DgsService and remove them if they meet all of the following criteria:

  1. Only connected on one side
  2. Does not have an auxiliary equipment attached to their terminal
  3. Doesn't belong to a feeder
  4. If it belongs to a feeder, it must not contain the feeder head terminal
    This should improve visual cluttering of auto-drawn internal schematics diagram, and remove unnecessary noise on the geographic diagram.

Return to Full List of Processors

StarPointProcessor​

Required: distributionTransformerConfig The processor takes the rGround and xGround value from the configuration and assign them to every DTX.

Return to Full List of Processors

SubstationEdgeElmTermSortingProcessor​

The processor sorts all ElmTerm that is associated with StaCubic belonging to ServiceElement within an ElmSubstat into the ElmSubstat by assigning the the ElmSubstat's fID to the ElmTerm's foldID.

Return to Full List of Processors

SubtransProcessor​

The processor cleans up the following subtrans related issues:

  1. Consolidate all circuit grid and all their equipment into their respective loop grid to lower grid clusters.
  2. If BSP substations exists in the model, all energy sources in non-BSP substations are disabled for load flow studies.

Return to Full List of Processors

TransformerProcessor​

The processor cleans up the following PowerTransformer related issues:

  1. Ensure Flanking ElmTerm of ElmTr2 has matching phase technology for its vectors
  2. Specific ElmTerm is assigned as Busbar to allow better power factory internal drawing
  3. Phase technology of ElmTerm and immediate StaCubic (LV side) is fixed up for different transformer functions (within the ElmSubstat for non-LV models which are processed by the LvElmTermProcessor
  4. Site type is updated for DTX for geographic diagram symbol assignment
  5. Set ElmTerm and ElmCoup post PowerTransformer neutral connection to be PHH1_N for phase consistency
  6. Adapting values from NER (Ground from CIM) as zone substation impedance

Return to Full List of Processors

TriSwitchProcessor​

The processor cleans up tri-switch formation for better geographic diagram clarity.

Return to Full List of Processors

VariantProcessor​

This is a processor to sort objects created in hosting capacity scenarios into variant stages (IntSstage).
