power-factory-exporter

Power factory exporter library - Maps network data from Cim to power factory compatible DGS format.

Starting ðŸš€â€‹

These instructions will allow you to get a copy of the project running on your local machine for development and testing purposes and/or running a docker container as well.

Mavenâ€‹

Add to your pom.xml:

com.zepben.powerfactoryexporter0.7.0

Usageâ€‹

For a basic example of using the power factory exporter refer to the demonstration code in file com.zepben.powerfactory.exporter.demo.Demo.kt

    val logger: Logger = LoggerFactory.getLogger("main")

    GrpcChannelFactory.create(ConnectionConfig(demoConfig.ewb.host, demoConfig.ewb.grpcPort, demoConfig.ewb.secureConnection)).use { channel ->
        val dgsGenerator = DgsGenerator(DgsGenerator.Config(), XlsxDgsWriter()) { NetworkConsumerClient(channel) }

        when (val result = dgsGenerator.generate(demoConfig.generationConfig)) {
            is DgsGenerationResult.Success<XSSFWorkbook> -> {
                writeFile(result.dgs, "output/model.xlsx")
            }

            is DgsGenerationResult.Failure<*> -> {
                logger.error("Failed to create model with the following errors:")
                result.errors.forEach { (m, t) -> logger.error(m, t) }
            }
        }
    }

### Workflow Overview ###
[![Where the library sits](/img/PFEWorkFlow.png)](/img/PFEWorkFlow.png)
1. User Request Power Factory Model by sending Model Config to Power Factory Model Builder
2. Model Config sent to Power Factory Exporter<br />
``
This library starts here
``
3. Request Network Model from Energy Work Bench
4. Receive Network Model from Energy Work Bench
5. Return DGS to Power Factory Model Builder<br />
``
This library ends here
``
6. Sent DGS to Power Factory Template Service
7. Return DGS to Power Factory Model Builder
8. Return DGS to EWB App Server


### Library Concept ###
[![Where the library sits](/img/LibraryConcept.png)](/img/LibraryConcept.png)
Given Meta data and a list of equipment containers, this library will map CIM data of equipments belonging to the list
of equipment containers to Power Factory Elements.

#### FinalCleanUpProcessor ####
The processor does a final pass on the model to clean up potential errors and reduce cluster. <br />

2. Final pass to ensure elements are sorted into the correct folder<br />
3. Remove Current transformers that were attached to elements removed during clean-up/post process<br />
4. Remove unused StaCubic and ElmTerm from grounding switches and assign correct objBus to staCubic that is in use<br />
5. Remove all Auxiliary equipments from zone sub because they are just clutter<br />
6. Remove any Stacubic (Terminals) that are not being used by any equipment<br />
7. Remove any equipment that is not connected to anything<br />
8. Remove and Elmterm(Connectivity node) that has nothing attached to it<br />
9. Disable all isolation transformer and auxiliary transformer in substations

1. Consolidate all circuit grid and all their equipment into their respective loop grid to lower grid clusters.<br />
2. If BSP substations exists in the model, all energy sources in non-BSP substations are disabled for load flow studies.

#### TransformerProcessor ####
The processor cleans up the following PowerTransformer related issues:<br />
1. Ensure Flanking ElmTerm of ElmTr2 has matching phase technology for its vectors<br />
2. Specific [ElmTerm](#elmterm) is assigned as Busbar to allow better power factory internal drawing<br />
3. Phase technology of [ElmTerm](#elmterm) and immediate [StaCubic](#stacubic) (LV side) is fixed up for different transformer functions (within the ElmSubstat for non-LV models which are processed by the [LvElmTermProcessor](#lvelmtermprocessor)<br />
4. Site type is updated for DTX for geographic diagram symbol assignment<br />
5. Set [ElmTerm](#elmterm) and [ElmCoup](#elmcoup) post PowerTransformer neutral connection to be PHH1_N for phase consistency<br />
6. Adapting values from NER (Ground from CIM) as zone substation impedance
