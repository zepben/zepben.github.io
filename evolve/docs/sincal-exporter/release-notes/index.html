v2.9.0

Breaking Changes
  • None.
Enhancements
  • Added option to select the load input type in Sincal files (P and Q, P and power factor, or S and power factor).
  • Now able to read transformer utilisation from server payload and apply this to the fS field in Sincal.
  • More GIS types are treated as switches, to preserve these when they are exported: Fuses, Arc_Chutes, Live_Line_Clamps and Bridges will now all be treated as switches rather than nodes upon export.
  • Iso transformer tap step size now set to 2.5% by default.
  • When using the "Ignore Phasing Data" option, SWER network created downstream of SWER isolators will be on a random phase, rather than Phase L1.
  • The export type (DMS or GIS) is now saved as a setting.
  • There is a new basic check of phasing included - if two lines connect at the same node in an impossible way (for example, a node with only L2 and L1 connected), there will be a warning. However, this feature does not include all possible incorrect phasing checks.
Fixes
  • Regulators with no secondary network are able to be created.
  • The default time in the "power export" is now 16:00.
  • The output target will now be appended with .sin if it's missing and removing trailing whitespace.
  • Improved recognition of single phase elements with a neutral, so that these come into SINCAL as the appropriate single phase.
  • Distribution transformers now sense where the primary terminal is, rather than relying on the terminals coming in the correct order from the server.
  • Loads now inherit their phase properly from server data.
  • Switches on lines that are combined with other lines are preserved better.
  • Improved setting of primary and secondary voltage on isolators.
  • A problem with switches being placed at the incorrect end of lines has been resolved.
Notes
  • None.

v2.8.0

Breaking Changes
  • None.
Enhancements
  • HTTPS support for both the network data source and load data source.
  • Client authentication using Auth0, when EWB is setup with authentication.
  • Network and load data source is saved in the user settings now.
  • Added settings and control to use Sincal Standard database for line data, where standard database reference is part of GIS payload.
  • Added option to create distribution transformers rather than single loads where only MV data is included in JSON payload.
  • Use the voltage from the JSON payload, rather than overwriting all voltages with single configured voltage.
  • Create support to have multiple infeeders at different voltages within the zone substation.
  • Added support for all transformer vector groups that Sincal currently supports.
  • Added additional checks to prevent isolating transformers being turned into SWER isolators when they are not (checks based on further isolators downstream, and downstream network voltage).
  • Support for SWER regulators.
Fixes
  • UI fix for some controls not disabling during exporting of model.
  • Minor UI fix for the message box UI to stay on top of the application.
Notes
  • None.

v2.7.0

Breaking Changes
  • None.
Enhancements
  • Import from GIS based data sources is now possible.
  • Phase data (from GIS based data only) is now available in exported files.
  • User settings can now be stored in profiles, rather than as a single set of options only.
  • Load data can be on a separate server to network data if required.
  • Improved detection of isolators that are not SWER isolators.
  • Additional checks for suspect cable data within the cable dictionary.
  • Better default scaling of co-ordinates to make inserting map backgrounds easier.
  • Include cable type information in the Sincal 'Conductor Info' field.
Fixes
  • Regulator primary and secondary sides should now be assigned properly in all cases.
  • Regulator sites including fault indicators are now collapsed properly.
  • Infeed bus name is no longer blank if feeder circuit breakers do not include the substation name.
Notes
  • Bringing in GIS based data sources and phase data has required numerous changes to the internal processing of data.

v2.6.1

Breaking Changes
  • None.
Enhancements
  • Made the rewiring of ISO transformers to look for direction not single/multi phasing.
  • Altering default text position to be offset from nodes; allows easier manipulation of network data for end users.
  • Now only adding broken bridges if they are closed.
  • Writing Flag_LineType for underground conductors.
  • Additional Sincal layer for 0 length conductors.
  • Style underground conductors in Sincal with broken lines.
  • Energy profile load flow.
  • Transformer name plate rating load flow.
  • Remove "cancelled by user" Prompt.
  • Prompt when export has finished.
  • Updated default sincal template with new colors for the layers.
  • Log date is now Australian date format.
  • Geographic location of nodes are written to make it easier for inserting background map.
  • Prepending of some nodes with FN to make it easier for fitting the diagram to the background map.
Fixes
  • Corrected the handling of conductors with more than two terminals which resulted in excess buckle points being created.
  • Sincal exporter failing to deal properly when any feeder with a HV customer metering point and HV customer are connected in series.
  • Fixed an issue where the log is scrolling up and down with the export finished.
Notes
  • None.

v2.5.0

Breaking Changes
  • None.
Enhancements

  • Missing graphics can now be interpolated.
