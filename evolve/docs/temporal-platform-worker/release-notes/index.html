Release History​

VersionDate
0.2.002 October 2025

Breaking Changes​

  • Workflow arguments encoded as Json are now expected to have all field names transformed to all lowercase. e.g. the field "calibrationName": needs to be encoded as "calibrationname":
  • The tap calibration related workflows and activities now expect the PQV data in the load database to be stored in UTC rather that local time.
  • PopulateCalibrationTable.runForFeeder now will only calculate taps if transformer tap settings are not provided to the workflow.
  • Calibration workflow now uses the global modelConfig default values.

New Features​

  • Add Sincal Forecast Workflow

Enhancements​

  • Calculation of tap settings will now take into account the existing transformer tap settings from the CIM network.

Fixes​

  • Fix memory allocation of HCInputDbImporterWorkflow. Added batchSize to the hcinputdbimporter config for further tuning if needed.
  • Create ingestor job with ttlSecondsAfterFinished set to 5 minutes. This may improve the reliability of uploading ingestor logs
  • Fixed issue with GenerateCalibrationMeterGroupImpl and PopulateCalibrationTableImpl not handling feeders with EnergyConsumers but no CPI's.
  • Fixed issue with CalibrationRunImpl not writing out calibrated tap settings when transformerTapSettings was not supplied.

Notes​

  • Updated to temporal-platform-definitions to 0.4.0, com.zepben.commons and com.zepben.ewb v1.1.0\n\n---\n
