EDNAR Reporting Service

The EDNAR Reporting Service exposes an API which allows EDNAR server to:

  • Generate NAR PDF.
  • Generate NAR attachment PDF.
  • Generate customer list PDF.
  • Generate the switching instruction PDF.

Required Software

  • At least Windows 7 operating system with SP1 or above is required
  • .NET Framework 4.6.1 or higher
  • Aspose.Total
  • DevExpress.Reporting and DevExpress.WPF

Installing EdnarReportingService

info

You need administrator privileges to install the service.

  • Login to the windows machine in which EdnarService needs to be installed.
  • Create a folder EdnarReportingService.
  • Copy all the EdnarReportingService binaries from the .zip file of the release to this folder.
  • Copy the Aspose license file to folder EdnarReportingService.
  • Change the config file to make sure it is correct in the running environment.
  • Run Powershell as an administrator and use it to navigate to the EdnarReportingService directory.
  • Run .\ednar-reporting-service.exe install and wait for a message to say that it was successfully installed.
  • The EdnarReportingService windows service is now installed.
  • Go to the Services console in windows.
  • Make sure the EDNAR Reporting Service service is running. If not, start it.
  • Check the properties of the EDNAR Reporting Service service to ensure that it is configured to start automatically.

Uninstalling EdnarReportingService

info

You need administrator privileges to uninstall the service.

  • Login to the windows machine in which EdnarReportingService is installed.
  • Run Powershell as an administrator.
  • Navigate to \EdnarReportingService.
  • Run the command .\ednar-reporting-service.exe uninstall
  • You should get an uninstall successful message.

Configuring EdnarReportingService

  • Navigate to C:\EdnarReportingService.
  • Open the file ednar-reporting-service.exe.config in the text editor of choice.
  • Update the configuration items as specified below. Save and close the file. Restart EdnarReportingService for the new configuration to take effect.
<add key="EdnarReportingServicePort" value="8010" />
<add key="LogFileRelativePath" value="Logs\" />
<add key="ServerBaseUrl" value="http://+:" />
<add key="LocalRequestOnly" value="true" />
<add key="ReportTemplateFile" value="report-template\\schedule_job_report" />
<add key="DocumentDefinitionFile" value="report-template\\PCA_SCHEDULED_JOB_HEADER.xml" />
<add key="CustomersExcelTemplateFile" value="report-template\\CustomersReportTemplate.xlsx" />
<add key="CustomersExcelTemplateFilePON" value="report-template\\CustomersReportTemplatePON.xlsx" />
<add key="CITIHandCardWordTemplateFile" value="report-template\\CitiPower Hand Card.doc" />
<add key="PCORHandCardWordTemplateFile" value="report-template\\Powercor Hand Card.doc" />
<add key="MaxConcurrentRequestsLimit" value="2" />
<add key="MaxQueueRequestsLimit" value="8" />
<add key="AsposeLicense" value="Aspose.Total.lic" />
<add key="EwbBaseUrl" value="http://localhost:9000/ewb" />
<add key="CdmBaseUrl" value="http://localhost:9988/customers" />
<add key="CdmUsername" value="xxx" />
<add key="CdmPassword" value="yyy" />
<!-- Use ONE of these two connection string formats depending on database format (Oracle or MS SQL Server) -->
<!-- SQL Server: -->
<add name="EdnarDB" connectionString="Server=my-sql-server;Database=ednar;User Id=ednar;Password=password;" />
<!-- Oracle: -->
<add name="EdnarDB" connectionString="Data Source=(DESCRIPTION=(ADDRESS=(PROTOCOL=TCP)(HOST=192.168.30.129)(PORT=1521))(CONNECT_DATA=(FAILOVER_MODE=(TYPE=select)(METHOD=basic))(SERVER=dedicated)(SERVICE_NAME=nms)));User Id=EDNAR;Password=password;" />
<add name="DMSOracleDB" connectionString="Data Source=(DESCRIPTION=(ADDRESS=(PROTOCOL=TCP)(HOST=192.168.30.129)(PORT=1521))(CONNECT_DATA=(FAILOVER_MODE=(TYPE=select)(METHOD=basic))(SERVER=dedicated)(SERVICE_NAME=nms)));User Id=enmac;Password=password;" />
<add name="PonOracleDB" connectionString="Data Source=(DESCRIPTION=(ADDRESS_LIST=(ADDRESS=(PROTOCOL=TCP)(HOST=192.168.30.129)(PORT=1521)))(CONNECT_DATA=(SERVICE_NAME=nms)));User Id=po_connect;Password=DEV!43po_connect;" />
Config ItemDescription
EdnarReportingServicePortEdnarReportingService port number
LogFileRelativePathPath of the log files relative to the deployment directory. A path of Logs\ will create the log file in C:\EdnarService\Logs if the deployment directory is C:\EdnarService. Log file is created in text format and the rolling interval is set to a day.
ServerBaseUrlBase server URL that will be used by the client to send http requests e.g. https://+:.
LocalRequestOnlyThe flag to set if only the local request is valid.
ReportTemplateFileThe switching instruction report template file path including the filename.
DocumentDefinitionFileThe switching instruction document definition XML file path including the filename.
CustomersExcelTemplateFileThe customers report template file, including the path name.
CustomersExcelTemplateFilePONThe customers report template file that will be used for PON, including the path name
CITIHandCardWordTemplateFileThe hand card template of CitiPower, including the path name
PCORHandCardWordTemplateFileThe hand card template of Powercor, including the path name
MaxConcurrentRequestsLimitThe maximum count of reports that will be created at the same time. This limits the memory consumption of the service.
MaxQueueRequestsLimitThe maximum count of requests that will be put into queue once MaxConcurrentRequestsLimit has been reached. This affects users having to wait for a long time for the report to be generated. Instead, once the limit has been reached, the service will immediately return 503 Service Unavailable.
AsposeLicense (optional)The Aspose license file, including the path name. When not specified, the license should be on the same directory with the executable files.
EwbBaseUrlHTTP base url of the Energy Workbench Server. Note: /ewb is not defined in the service. Hence; must be included in the base url.
CdmBaseUrlHTTP base url of the customer data management service.
CdmUsernameUsername for authentication with customer data management service.
CdmPasswordPassword for authentication with customer data management service.
Connection String - EdnarDBSpecify the connection string for the EDNAR database, can be Oracle or MS SQL.
Connection String - DMSOracleDBSpecify the connection string for the DMS database in oracle.
Connection String - PonOracleDBSpecify the connection string for the PON database in oracle.
note

The port number needs to be consistent with the EDNAR app server configuration.
