DMS Integration

EDNAR uses the PowerOn WPM SOAP interface to communicate with the PowerOn Fusion DMS or PowerOn Advantage ADMS (simply referred to as the DMS for this documentation). The DMS functionality is now contained in a separate DMS Agent application.

Direct Mode

Client Server Mode

Functionality

  • When a NAR is moved to "Accepted" state in the NAR DMS Schedule lifecycle, EDNAR automatically triggers a request to the DMS agent to create a schedule or job. The DMS Agent maps the NAR fields as defined in the EDNAR DMS Mapping file and sends them to the DMS with the request to create a schedule or a job, using either the CreateSchedule or CreateJob WPM SOAP requests.

  • On receiving a request to create a schedule from the DMS Agent, the DMS creates a planned Switching Schedule or Job at the "Created" status with all the appropriate Header information populated, as per the above EDNAR DMS mapping file.

  • On auto creation of the DMS Schedule the DMS job reference id ("J" number) is sent back to EDNAR.

  • Additional functionality using other SOAP interfaces is also available, including creating a Schedule Pin on the DMS Diagram, creating documents/permits against jobs, and populating LV Switching Steps from the NAR as text steps in the DMS switching schedule.

  • As the job/schedule progresses through its lifecycle in the DMS, the WPM Outbound SOAP interface triggers requests from the DMS to the EDNAR DMS Agent with information about the DMS job status. These requests are processed by the agent and passed on to the EDNAR App Server, where NARs are transitioned to align with the state of the DMS job, and some additional DMS Switching Details are also recorded against the NAR.

Configuration

The DMS endpoint is configured in the EDNAR API configuration file. More information on configuration can be found on API Server Configuration.

Each customer will need to supply a mapping of EDNAR fields to the corresponding DMS job header fields. Contact Zepben for more information on this.
