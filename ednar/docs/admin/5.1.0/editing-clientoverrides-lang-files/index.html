Editing clientOverrides lang files

This guide goes over the steps to follow to edit the clientOverrides.json lang file. This file is used to change specific pieces of text in the EDNAR UI from their default values, e.g. changing NMI to ICP for New Zealand customers.

note

This guide is written for people with little to no knowledge of JSON format

1. In the EDNAR web client folder, go to the "lang" directory. Open one of the English lang files (e.g. en.json) in Notepad++ (or similar). Hit Ctrl+F to find the text key(s) you want to change.

2. Once you've found the key to change, you'll need to copy the name of the JSON object that key lives in. This is the "section" name (e.g customerNotificationTask).

3. In the clientOverrides.json file, paste the object name between the braces ({}), keeping the double quotes intact. Then add a colon and another set of braces after the object name. e.g.

{
   "customerNotificationTask": {}
}

4. Back in the en.json lang file, copy the key and value you wish to change. In the clientOverrides file, paste them between the braces you added after the corresponding object name (this should be across multiple lines for readability) e.g.

   {
   "customerNotificationTask": {
      "saidiMinutes": "SAIDI Minutes"
   }
}

5.Now edit the value (right side of the colon) only to whatever text you want to display. If you edit the key (left side of colon) the change will not work. e.g.

{
   "customerNotificationTask": {
      "saidiMinutes": "CMOS Value"
   }
}

6. To add additional keys under the same object, make sure that each line has a comma between them. The last line in an object doesn't need a trailing comma. e.g.

{
   "customerNotificationTask": {
      "saidiMinutes": "CMOS Value",
      "outageType_SHORT_SHUTDOWN": "Generator Shut Down - 15 minutes",
      "outageType_MAYBE": "Target Outage"
   }
}

7. To add keys that belong to different objects (e.g. not under "customerNotificationTask"), you'll need to make sure that you include commas between each object as well. e.g.

{
   "customerNotificationTask": {
      "saidiMinutes": "CMOS Value",
      "outageType_SHORT_SHUTDOWN": "Generator Shut Down - 15 minutes",
      "outageType_MAYBE": "Target Outage"
   },

    "nar": {
        "primaryEquipmentContainer": "Primary Feeder or Zone"
    }
}

8. Once you've made the changes, as long as the clientOverrides is located in the lang folder, the changes should be reflected in the UI when you reload the EDNAR web page.
