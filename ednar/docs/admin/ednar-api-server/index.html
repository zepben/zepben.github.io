EDNAR API Server

The EDNAR Web API server exposes many end points which are consumed by the EDNAR web client to provide functionality to the end user. When authentication is configured, all requests to this server must contain a valid JWT.

EDNAR Web API server is hosted on Vertx web server (https://vertx.io)

Required Software

  • Windows operating system (Windows Server 2012)
  • Java SE 11 x64

Starting and Stopping the Web Server

Start

  • Open Powershell
  • Navigate to the folder that contains the EDNAR Web API binaries
  • Run the command java -jar ./ednar/ednar.jar -c ./ednar/config.json

Stop

  • Focus on the Powershell window.
  • Press the key combination Ctrl + C.
  • Close the Powershell window.

Configuration

Please see EDNAR API Server Configuration

Database Object Management

EDNAR uses Ebean (https://ebean.io/) as the Object Relational Mapper. It uses Liquibase (http://www.liquibase.org/) for database schema creation and migration.

When EDNAR starts, it runs a set of liquibase scripts to perform schema creation and migration. Unfortunately, Oracle doesn't provide transactions for DDL statements (other RDBMS claim to support this). This means there is a potential for the DB to end up in an incomplete state if for some reason a migration fails. In this case, restoring from a DB backup or manual intervention will be required.

Physical Location on Server

The physical location of EDNAR Web API binaries is C:\EDNAR\Server. A different location can be chosen if desired. However, this document assumes the use of this location.

Deployment Steps

  • Stop server
  • Backup Oracle database
  • Backup existing files from previous version
  • Extract files from new release package into the Web API folder
  • Make any appropriate configuration changes as per release notes
  • Run any database scripts as per release notes
  • Start server

Rollback Steps

  • Stop server
  • Restore Oracle database from database backup.
  • Restore from backup files from previous Web API version
  • Start server
