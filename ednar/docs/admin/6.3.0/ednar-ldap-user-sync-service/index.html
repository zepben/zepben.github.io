EDNAR LDAP User Sync Service

The EDNAR LDAP User Sync Service is a standalone Java application that binds to an AD using LDAP, polls user data from both the AD and EDNAR App Server, then reconciles them and maps them into a http request that is sent to EDNAR to create and/or update those users.

Running the LDAP User Sync Service​

You can use the following command to run the LDAP User Sync Service. The config file format is outlined below.

java -jar ldap-user-sync-service.jar -c <path to config.json>

Sample Config​

{
    "ldapServer": {
        "ldapHostName": "localhost",
        "ldapPort": 1389,
        // base DN to start searching for users
        "baseDn": "dc=example,dc=com",
        // DN of user to bind with
        "bindDn": "cn=read-only-admin,dc=example,dc=com",
        "bindPassword": "password for above bindDn"
    },
    "ldapMappings": {
        "groupNamesMappings": {
            "cn=readers,ou=users,dc=example,dc=org": "Role Name in EDNAR",
            "ou=mathematicians,dc=example,dc=com": "Different role name in EDNAR",
            "ou=chemists,dc=example,dc=com": "Requester",
            "ou=scientists,dc=example,dc=com": "Admin",
            "ou=italians,ou=scientists,dc=example,dc=com": "Super User"
        },
        "groupMembersAttrKey": "uniqueMember",
        "userAttrsClaimsMappings": {
            "username": "uid",
            "firstName": "givenName",
            "lastName": "sn",
            "email": "mail",
            "mobile": "telephoneNumber",
            "phone": "telephoneNumber",
            "fax": "missing",
            "position": "position",
            "depot": "depot",
            "organisation": "o"
        }
    },
    "ednarServer": {
        "host": "localhost",
        "port": 5501,
        "protocol": "HTTP",
        "clientId": "<EDNAR API client ID>",
        "clientSecret": "<EDNAR API client secret>",
        "verifyCertificate": false
    },
    "timeUnit": "SECONDS",
    "retryPeriod": 35,
    "initialDelay": 4
}

Sample Request to POST /api/v0/config/users/bulk endpoint on EDNAR App Server​

 {
  "users": [
    {
      "username": "testUser1",
      "isDisabled": true,
      "firstName": "John",
      "lastName": "Smith",
      "email": "email@email.com",
      "mobile": "mobile123",
      "position": "job title goes here",
      "roleNames": ["Requester"]
    },
    {
      "username": "TestUser2",
      "isDisabled": false,
      "firstName": "Bob",
      "lastName": "Brown",
      "email": "testuser2@test.com",
      "mobile": "mobile123543",
      "position": "job title goes here",
      "depotName": "Albury",
      "roleNames": []
    }
  ],
  "updateRoles": true
}

  ]
}
