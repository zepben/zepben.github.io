EDNAR API Server Configuration

This page describes how to configure and run the EDNAR WEB API Server.

Prerequisites

  • Java 11
  • If using Oracle, a user already created with permissions to create schema objects and run various DDL statements.

Configuration

Configuration for the EDNAR server is specified in a JSON file.

KeyDescription
authObject holding authentication config
Refer to auth options below.
databaseObject holding database config
Refer to database options below.
dmsObject holding DMS config
Refer to dms options below.
ewbObject holding EWB config
Refer to ewb options below.
httpObject holding HTTP config.
Refer to http options below.
ponObject holding PoN config
Refer to pon options below.
cdmObject holding CDM config
Refer to cdm options below.
smtpObject holding SMTP config
Refer to smtp options below.
reportsObject holding report config
Refer to report options below.
Auth Config
OptionDescriptionRequiredDefault
typeThe type of config. Currently only the value Layer7 is supported.Yes
loginUrlThe Layer7 endpoint url for login.Yes
validateUrlThe Layer7 endpoint url for validating a token.No
verifyHostFlag for host verification for https connections on above URLsNotrue
trustAllCertificatesFlag to trust all https certificates for above URLsNofalse
Database Config
OptionDescriptionRequiredDefault
driverThe JDBC driver classYes
urlThe JDBC connection stringYes
userThe user to connect to the databaseNo
passThe password to connect to the databaseNo
DMS Config
OptionDescriptionRequiredDefault
urlThe SOAP endpoint of PowerOn Fusion listening for WPM SOAP messages.Yes
connectTimeoutThe timeout in ms before the SOAP client deems it failed to connect to the server.No10000
readTimeoutThe timeout in ms before the SOAP client deems the server failed to respond.No10000
listenerPortThe port the EDNAR will listen on for outbound WPM SOAP messages.Yes
listenerPathThe path EDNAR will accept outbound WPM SOAP messages.Yes
listenerLogRequestsBoolean flag that will enable SOAP request access log.Nofalse
EWB Config
OptionDescriptionRequiredDefault
hostThe host name of the EWB server.Yes
portThe port the EWB server is listening on.Yes
basePathA path that will be prepended to all calls to EWB.Yes
HTTP Config
OptionDescriptionRequiredDefault
portThe port for the server to listen on.Yes
webrootThe path to the directory holding the application static assets.Yeswebroot
staticAssetCachingWill cache static assets in memory if set to true. This improves performance but means that the server needs to be restarted if static assets change.Notrue
httpsEnables https by specifying the certificate type. Supported types: pem.No
certPathThe file path the the https certificate when https is pem.No
keyPathThe file path the the private key file when https is pem.No
csrfSecretSecret used to generate crytographically secure CSRF tokens.Yes
protocolsList of transport security protocols to enable. Valid values are "TLSv1.1", "TLSv1.2".No["TLSv1.2"]
logRequestsLogs all http requests to the log when set to true.Nofalse
apiOnlyDisables serving of static assets and only enables API routes.Nofalse
PoN Config
OptionDescriptionRequiredDefault
hostThe host name of the PoN server.Yes
portThe port the PoN server is listening on.Yes
CDM Config
OptionDescriptionRequiredDefault
hostThe host name of the CDM server.Yes
portThe port the CDM server is listening on.Yes
usernameUsername for authentication of EDNAR against CDM service.Yes
passwordPassword for authentication of EDNAR against CDM service.Yes
SMTP Config
OptionDescriptionRequiredDefault
hostThe host name of the SMTP server.Yes
portThe port the SMTP server is listening on.Yes
userThe user to connect to the SMTP serverNo
passThe password to connect to the SMTP serverNo
Reports Config
OptionDescriptionRequiredDefault
narReportPdfUrlThe report server endpoint url for getting the nar report pdf.Yes
narHandCardPdfUrlThe report server endpoint url for getting the nar hand card pdf.Yes

If any paths in the configuration are given as relative paths, they will be relative to the current working directory.

Sample Configuration File

{
  "http": {
    "port": 443,
    "webroot": "./webroot",
    "https": "pem",
    "certPath": "/path/to/the/cert.pem",
    "keyPath": "/path/to/the/key.pem",
    "logRequests": true
  },

  "database": {
    "driver": "oracle.jdbc.driver.OracleDriver",
    "url": "jdbc:oracle:thin:@localhost:1521:xe",
    "user": "ednar",
    "pass": "Password1"
  },

  "auth": {
    "type": "Layer7",
    "loginUrl": "https://my-layer7.invalid/sso/jwt/login/ednar",
    "validateUrl": "https://my-layer7.invalid/sso/jwt/login-validation/ednar"
  },

  "smtp": {
    "host": "smtp.myhost.invalid",
    "port": 25,
    "user": "username",
    "pass": "password"
  },

  "dms": {
    "url": "http://pof-server/enmac/SOAP"
  },

  "ewb": {
    "port": 9000,
    "host": "my-ewb-server",
    "basePath": "/ewb"
  },

  "pon": {
    "port": 80,
    "host": "my-pon-server"
  },

  "cdm": {
    "port": 8081,
    "host": "my-cdm-server"
  },

  "reports": {
    "narReportPdfUrl": "http://report-server/report",
    "narHandCardPdfUrl": "http://report-server/hand-card"
  }
}

Run

Command Line Arguments
FlagValid ValuesRequiredDefault
-confPath to the configuration fileYes

To run the server, execute the following command at a terminal:

java -jar <path to ednar jar> -c <path to config file>
