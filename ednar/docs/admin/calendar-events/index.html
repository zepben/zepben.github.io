Calendar Events

Calendar Events Home​

Calendar Events can be viewed and managed using the admin app by navigating to the Event Management page from the top menu.

Creating Events​

Clicking on the + Create Event button at the top of the screen will take you to the Create Event page.

From here, you can enter the details of the calendar event to be created in the EDNAR system. The following fields are available:

  • Name: The name of the event. This will display within the EDNAR system.
  • Types: The type of the event. This dictates the colour that the event will display as in the EDNAR system, as well as whether the event will affect lead times, e.g. a public holiday. These must be configured in the EDNAR database. See the Config Schema page for more details.
  • Start Date: The start datetime of the event. Note that event types flagged as Public Holidays can only extend for a single calendar day.
  • End Date: The end datetime of the event. Events that are not public holidays can extend across multiple days.
  • Scope: Events can be created as "Global" events (default), or they can be limited to particular Depots or Regions within the system. Only one depot or region may be specified per event.
  • Description (optional): Additional details of the event. This will display when a user hovers over the event in the EDNAR system.

Once these fields have been populated, pressing the Create Event button in the top right will create and save the event.

Bulk CSV Upload Calendar Events​

EDNAR supports uploading CSV files containing one or more calendar events. This allows users to upload events in bulk to save each event being created one at a time.

To upload a CSV, press the Import from CSV button at the top of the Event Management landing page. This will prompt you to select a CSV file from your files to upload. The CSV must follow the below template. If the upload is successful, you will receive the count of events created and events updated. Events will be updated if they have the exact same name, type and depot or region, otherwise they will be created new. Events in the CSV with all fields identical as an existing calendar event will be ignored.

Bulk CSV Upload File Format​

START_DATE_TIMEStart time of the event. Must be in one of the required formats, listed below.Yes
END_DATE_TIMEEnd time of the event. Must be in one of the required formats, listed below.Yes
NAMEName of the eventYes
TYPEType of the event. Must match exactly with one of the configured Event Types in the EDNAR system.Yes
DEPOTDepot the event should apply to, if applicable. Must match exactly with one of the configured Depot names in the EDNAR system.No

info

Only the DEPOT or the REGION column can be populated for each row. If neither is populated, the event will be a Global event.

Supported Datetime Formats​
  • D/MM/YYYY hh:mm
  • DD/MM/YYYY hh:mm
  • D/MMM/YYYY hh:mm
  • DD/MMM/YYYY hh:mm
  • YYYY-MM-DD hh:mm:ss.SSSSSS

Updating and Deleting Calendar Events​

Calendar events can be edited or deleted via the Event Management landing page. There is an Edit and a Delete button to the right of each event listed.
