EDNAR DMS Agent

The EDNAR DMS Agent is a standalone Java application that acts as the middleware between the EDNAR App Server and the DMS. Details on the functionality and workflows can be found on the DMS Integration page.

Configuration

The DMS Agent requires configuration on the EDNAR App Server and on the EDNAR DMS Agent itself. The App Server configuration required can be found in the API Server Configuration page.

The DMS Agent can either be configured in Direct Mode or Client-Server Mode. In Direct Mode, only one instance of the DMS Agent is required. In Client-Server mode, there are 2 instances of the DMS Agent - one acting as the Server, and one acting as the Client. All three modes have different configurations.

To use the DMS Agent, an API Client must be created that the Agent can use for authentication. This API Client requires the UPDATE_FROM_DMS privilege for all applicable access types. More information on creating API Clients can be found on the API Clients page.

Configuration is specified in a JSON file.

Running the DMS Agent

The EDNAR DMS Agent can be run in much the same way as the other EDNAR Java applications. To run the agent in any of the modes, the following command can be used:

java -jar ednar-dms-agent-VERSION.jar -c <path to config.json>
