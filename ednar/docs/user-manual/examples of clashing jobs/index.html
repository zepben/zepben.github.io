Examples of Clashing Jobs

The simplified schematic below represents 4 different HV feeders and 2 Zones.

  • HV Feeder BPM3B7 has feeder BPM3B2 as an adjacent feeder, and visa versa.
  • HV Feeder BPM3B7 also has feeder OPM3B4 as an adjacent feeder, and visa versa
  • HV Feeders OPM3B3 and OPM3B4 are in the same Substation (OPM)
  • HV Feeders BPM3B2 and BPM3B7 are in the same Substation (BPM)

This set of Feeders will be used in the discussion about how clash detection works.

The diagram above is a simplified schematic diagram â€“ how it would look in the EDNAR maps is illustrated below. In the top screenshot, NAR 696 has apparatus to be covered on Feeder BPM3B2. In the bottom screenshot, we have some apparatus to be covered on Feeder BPM3B7

In the screen shot below, we have NAR 697 open in the browser, and we can see that NAR 697 has one or more apparatus on feeder BPM3B7 â€“ this is one of the feeders in the diagram above. This feeder is adjacent to feeder BPM3B2. There is an open point between these feeders that can be closed to electrically connect these two feeders.

Any NARs that have apparatus to be covered on these two feeders will be included in the check for clashing NARs. The other test for clashing is when the work is scheduled to take place for the NAR.

In the example below, we have NAR 697 open in the browser, and we have run a clash detection â€“

  • NAR 701 is marked as Green, because it is on an adjacent feeder, but is occurring before the scheduled date and time of the NAR we are running the test for.
  • NAR 696 is marked as Red, because it is on an adjacent feeder and overlaps with the date and time with the NAR we are running the check on. In this situation, the clash must be acknowledged before the NAR can progress in its life cycle. Note that the NARs must overlap in both date and time of day for this to come up as a red clash.
  • NAR 698 is marked as Blue, because it is on an adjacent feeder, but is occurring after the scheduled data and time of the NAR we are running the test for.
