Rescheduling NARS

When a NAR cannot go ahead on the planned date, its possible to re-schedule the NAR. A NAR can only be re-scheduled if it is in any of the NAR accepted states and switching has not already commenced in the DMS schedule associated with the NAR.

LINK NAR accepted to 4.3

When the NAR is re-scheduled, a reason and Comment must be supplied.

If successful, the NAR will transition to the “Rescheduling state’. In this state only the following aspects of the NAR can be edited:

  1. The start and end date/time
  2. Whether or not there is a customer outage.
  3. Editing of notification tasks, including the creation of additional tasks.
  4. Network operator details in any Authorities

When the necessary changes have been made to the NAR, the user can “submit” the NAR again via the submit icon, as illustrated below.

Once the NAR has been submitted, it will go into the “Resubmitted” state. From this state, the NAR can be accepted as illustrated in the screen shot below.
