Customer Notification

Once the built-in Customer Notification question has been marked "Yes" in the Application Tab, the Customer List tab will become available.

Once this tab is opened, there will be a blank Customer Notification Task to fill out. If more than one customer notification task is required (e.g. for separate outage times), click the blue Add Task button at the bottom of the screen.

1. If the outage start and end times are different to those of the NAR, these can be selected here, otherwise the Sync to calendar values button will align the outage dates with the NAR dates. Note that the outage date times must be within the NAR date times - ineligible dates will be greyed out in the date selector.

2. A description of the notification task should be included here. This is currently mandatory.

3. Select the outage type and the notification delivery method here. Refer to your organisation for their policies on these. If your organisation uses an external outage viewer tool, you can confirm that it has been used via the Checked in Outage Viewer By: field.1

4. There are several ways to add affected customers to a notification task. The first is by using selected assets to run an isolation trace that will identify the affected customers. Isolation Points from Access Authorities can be used in the top tab - either click the blue asset name to use it, or press Add All to use all the isolation points.

In the middle tab, additional isolation points can be included for customer trace. Isolation points added here will not be stored against the NAR anywhere.

In the bottom tab, additional individual NMIs (ICPs in New Zealand) or Supply Points can be used to add customers by entering them into the correct box. When entering multiple NMIs or supply points, they must be separated by commas. These can also be added once the customers are generated.

Selected source assets are displayed in grey. Press the X to remove them, or once finalised press the Generate Customer List button and the list of affected customers will appear.

1. Information about the total number of customers will be included here, including customers per transformer, sensitive customers and affected zones/feeders.

2. Additional NMIs and Supply Points can be used to add customers to the list here. See above.1

3. To search/filter the customer list, adjust the dropdown on the right to select which field to filter, then enter the search term in the text box.

4. To sort the customer list table, click on the column title. Refer to your organisation for information about each column.1

5. Click on a customer to view more details about them.1

6. To remove individual customers from the notification task, click the X on their row. Note that if the notification task has been ordered, then removing customers will trigger cancellation notices to be ordered for them.
