Authorities

Authorities represent the access authorities that will be issued to field staff allowing them to perform work on or near the network that is associated with the NAR.

An access authority is a physical or virtual document that is handed to field staff working on or near the network that describes where they are allowed to work and what sort of work they are allowed to do. These are sometimes also called “permits” or “access permits”.

These permits must be returned to the person who issued them before the network where the work was being undertaken can be returned to its normal operating state.

For example, if a field crew is replacing a distribution power transformer, they will almost certainly need the electricity distribution network supplying the transformer to be de-energised.

What that looks like in the real world, and how its represented in the EDNAR maps is shown in the side-by-side images below. Note that in this example, the low voltage network is not shown in the EDNAR maps (on the right-hand side). In some systems, the LV network will also be shown on the maps.

Whether the LV network is included in the maps or not will depend on whether that data is available in the asset data source system.

In this example, the authority will describe where it is safe to work, and what sort of measures have been taken to ensure the network remains dead.

To provide another example, there may be a job that involves tree trimming where the trees are close to the powerlines, but not touching them, as illustrated in the picture on the following page. In this case, the network will not be de-energized.

The authority issued to field staff will let them know this. There will be work practices used to help ensure that the tree branches do not come too close or into contact with the lines while they are being cut.

Additionally, the protection devices (or circuit breakers) that are being used to detect faults may be modified to ensure they do not reclose if a tree branch does come into contact with the line.

Circuit breakers in electricity distribution networks will often re-close onto a fault when it is detected because tree branches will often fall onto the lines without anyone working nearby, which triggers the circuit breaker to trip, but then fall harmlessly to the ground.

Making the circuit breaker reclose avoids a lot of unnecessary outages.

The second tab of the NAR form is the Authorities Tab. This allows the population of the information required for authorities. Authorities are also referred to as Permits in some organisations.

1. The Type field allows for the type of Authority to be set. Clicking on the Type field opens a list of authority options that can be chosen from. Different authority types may have different mandatory and optional fields.

2. The Recall Time of the NAR can be set in this field. Clicking on right field allows for the timeframe, either hours or days, can be set. The left field allows for the number of hours or days to be chosen.

3. The Apparatus To Be Covered can be set by entering a search value into the Asset Name field. A list of assets will appear. Options from this list can be selected, once selected the asset will appear in the description field where the text can be edited. The Apparatus To Be Covered determines the Primary Zone of the NAR. This along with the Isolation Points and Protection Devices can also be added via the map.

4. The Work to Be Done field requires a description to be entered in the field.

5. As with choosing the Apparatus To Be Covered, Isolation Points for the NAR can be selected from a list after a value has been entered into the asset name field.

6. Network Operator Required? can be enabled by clicking the check box. Once the check box is selected the Issue Location needs to be entered. A name must be entered in the Issue To field. Issue Time and Cancel Time may also be required.

7. The Protection Devices are selected in the same way as Apparatus To Be Covered and Isolation Points are selected.

8. For Attachments to be added the drop down field must be clicked and the type of attachment must be selected. Clicking Select and Upload will open the file explorer which will allow a file to be attached. To view an attachment that has been added during the create or edit NAR process you must save the NAR after adding the attachment then you will be able to open the attachment.

9. To use the Special Requirements field the check box beside the title must be ticked. A description field will then appear. A description must be added or the check box unchecked.

10. Additional Authorities can be selected by clicking the add buttons at the bottom of the tab. The Access Authority type can be changed after these are selected.

However if someone is known to be working near the line, that behaviour needs to be disabled. The protection device that needs to have its auto-reclose disabled can be captured in the access permit.

Further examples of different access permits are provided in the table below. Note, these are examples only. The abbreviation (and display text) for these is configured to line up with terminology used by the DNSP.

AbbreviationStands forWhen used and side effects
EAPElectrical Access PermitThis would be used when workers require access to the network assets that would normally be live, after they have been de-energised and made safe to work on. For this type of access permit, isolation points and apparatus to be covered will be mandatory, and protection devices may also be required in the Authority.
A/RAuto reclose suppressionThis would be used when workers are working in proximity to the network, but the network is still live and protected by an auto reclose device.
Live LineLive line workThis would be used when workers are working on the actual network while it is live, using appropriate protective clothing and equipment. In this case, apparatus to be covered and protection devices will be configured in the Authority as mandatory fields, but no isolation points will be required.
LVIBSLow Voltage Isolation and Bonding StatementThis would be issued to a third party to indicate that we have isolated and earthed the low voltage supply.
SFTSanction for TestIssued to someone who will be performing tests on the network.
Switching OnlySwitching OnlyThis indicates there is no access to the network required, but there may be customers impacted.
VAVicinity AuthorityIssued for close proximity work, for example where a crane is operating close to the network.
SILVStatement of Isolation for a low voltage networkIssued to an external party to indicate that low voltage network supplying their equipment has been isolated.
