Specifying Multiple Isolation Points

These are used when the Applicant knows that some sort of back feed arrangement, or connection of a generator, will be used to limit the extend of an outage. In this case the Applicant will select the isolation points to define the extend of the isolated section, as illustrated in the following sequence of operations, which shows how an overhead line reconductoring job might be dealt with in EDNAR.

MapDescription
The applicant in this case has used the annotation tool to draw a polygon around the area of the network that is going to be impacted. The overhead line within this polygon should also be selected as “Apparatus to be covered”.
The switch at the start of the section of network being reconductored will be one isolation point.
When this is added as an isolation point, it will simulate the impact on the network, and if we zoom out, we can see that a large portion of the network will be made dead.
It may be decided that this impacts too many customers, and so additional isolation will be performed to limit the outage to only the overhead line being worked on. A Mid Span Isolator will be used to isolated the network at the end of the section of line being worked on
If we zoom out a little now, we can see that the only impacted customers will be those between the isolation points on the switch and mid span isolator.
If we zoom out even more, we can see where the back feed to reenergise will come from. The panel on the picture shows what this open point looks like when if you were to zoom into that part of the map.
