The Network Access Request

Network Access Requests, or NARs, are created by Applicants. Creating a NAR captures the following key information.

  • What work is being carried out,
  • When is the work being done and what network assets are impacted,
  • Which customers will be taken off supply, if any,
  • What sort of network access is required â€“ is there switching needed for the outage?
  • Who are the personnel involved with the NAR, and
  • Key questions relevant to the work being performed.

An Applicant supplies this information by interacting with the EDNAR client. A NAR may be saved as a draft and resumed later.

The Applicant can't submit a NAR until all the necessary information has been provided. What is required and what is optional depends on how the EDNAR application is set up.

A symbol is used in various places in the application to indicate whether the required information has been provided and is correct.

Information is missing â€“ it needs to be supplied by a user by interacting with the EDNAR application.

Information is incorrect, for example the dates picked for a customer outage are outside of the data range of the overall NAR. This needs to be corrected by a user interacting with the EDNAR application. In some cases, this symbol may appear because of a system level issue, in which case it will need to be fixed by a system administrator.

The information is correct and complete.
