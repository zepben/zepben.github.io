Switching Type

The images below show how the Switching Type can change depending on the Access Type.

Switching Type describes the complexity of the network access needed for the work to proceed.

It may also be referred to as the Job Type.

For example, a simple job may involve working near the network, with no actual interruption of the electricity supply. In this case, there is no switching involved, but access permits for a third party need to be issued. A complex job may involve multiple switching actions in the High Voltage network to de-energise the network before a permit to work on the network is provided to the field workers.

  1. Live Line Work – in this case work is carried out with isolating the network. For this type of work, isolation points would not be specified, but apparatus to be covered and protection devices would be.
  2. Distribution Backbone with Isolation – This isolation may involve a back feed from an adjacent feeder at an open point.
  3. Distribution Spur with Isolation – This isolation will not involve a back feed.
  4. Auto Reclose Suppression – This would be used where a crew is working in close proximity to the network, and Auto Reclose on an upstream protection device would be disabled.
  5. LV Only – Isolation points may be involved, but only at the LV level.
  6. ZSS Switching – Switching that is occurring within a Zone Substation (ZSS).

The Switching Type drives the following additional business logic in the application:

TypeDescription
Job GroupThis is used for Zone Mapping. Different Switching Types can have different Zone Mapping. The Zone Mapping associates the Zone a network asset is nominally supplied from with a planning area and control desk. The use of planning areas and control desks is described in the planning areas and control desks heading.
Default AuthoritySelecting the Switching Type can be configured to set the default Access Authority to display in the Authorities tab of EDNAR.Default authorities are not mandatory, and extra authorities can be added to a NAR.
Mandatory AttachmentsThe Switching Type sets the attachments that are required to be included in the NAR, if any.
Lead Time (Business Days)The Switching Type sets the minimum amount of business days required between submission of the NAR and the listed start date.If an applicant wants to submit a NAR that is within this lead time, that can still be done, however in this case, the NAR will go into a Pending Submission state, for special processing.
More Info Required ThresholdIf a NAR that has been submitted within the lead time is sent back to a draft state with a “more information required” request, it will be possible for the applicant to supply the requested information within a certain number of days and for the NAR to go back to the submitted state even though the NAR may be within lead time at the time of re-submission. Each Switching Type can have a different threshold value for this.
Mandatory Participant TypeThis is a participant type that is mandatory on the NAR. It can be different for different Switching Types.Additional participants can be added to the NAR.
Mandatory Summary QuestionsThe Switching Type defines which summary questions are mandatory – that is, that must be answered.
