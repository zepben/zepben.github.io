EDNAR API Server Configuration

This page describes how to configure and run the EDNAR WEB API Server.

Prerequisites​

  • Java 11
  • If using Oracle, a user already created with permissions to create schema objects and run various DDL statements.

Configuration​

Configuration for the EDNAR server is specified in a JSON file.

KeyDescription
authObject holding authentication config
Refer to auth options below.
databaseObject holding database config
Refer to database options below.
dmsObject holding DMS config
Refer to dms options below.
ewbObject holding EWB config
Refer to ewb options below.
httpObject holding HTTP config.
Refer to http options below.
customerDetailsObject holding customer details API config
Refer to customer details options below.
cdmObject holding CDM config
Refer to cdm options below.
smtpObject holding SMTP config
Refer to smtp options below.
reportsObject holding report config
Refer to report options below.
customerNotificationObject holding CustomerNotification config
Refer to customer notification options below.
peekObject holding Peek integration config
Refer to peek options below.

Auth Config​

Layer7 Auth Config​
OptionDescriptionRequiredDefault
typeSet to Layer7.Yes
loginUrlThe Layer7 endpoint url for login.Yes
validateUrlThe Layer7 endpoint url for validating a token.No
verifyHostFlag for host verification for https connections on above URLsNotrue
trustAllCertificatesFlag to trust all https certificates for above URLsNofalse
usersOnly when type is set to mock - optionally provide a list of user objects with passwordsNo
LDAP Auth Config​
OptionDescriptionRequiredDefault
typeSet to LDAP.Yes
ldapUrlThe LDAP resource url.Yes
ldapUserDnThe LDAP distinguished name of the user used to search for directory service users.Yes
ldapUserPasswordThe LDAP password of the user specified in ldapUserDn.Yes
ldapBaseDnThe LDAP base distinguished name for searching directory service users.Yes
ldapUserSearchFilterFormatThe LDAP syntax used to search for user. e.g. (&(ObjectClass=user)(UserPrincipalName=userName)), where userName will be replaced with the username logged-in in Ednar.Yes
ldapUserAttributesObject holding the user attributes mapping of LDAP.
Refer to ldapUserAttributes options below.		Yes
defaultRolesArray containing the role names as default roles to be assigned to new users.No
tokenSecretJWT signing algorithm secret. Minimum length of 32.Yes
tokenIssuerJWT token issuer.Noauth.ednar
tokenAudienceJWT token audience.Yes
tokenExpirySecondsJWT token expiry in seconds.No3600
ldapUserAttributes Config​
OptionDescriptionRequiredDefault
firstNameLDAP attribute name of first name.Yes
lastNameLDAP attribute name of last name.Yes
emailLDAP attribute name of email.Yes
mobileLDAP attribute name of mobile number.Yes
phoneLDAP attribute name of phone number.Yes
faxLDAP attribute name of fax number.Yes
depotLDAP attribute name of department.Yes
positionLDAP attribute name of work title.Yes
Mock Auth Config​
OptionDescriptionRequiredDefault
mockSet to trueYes
expirySecondsSeconds before the user will be logged out and must sign in againNo3600
usersProvide a list of user objects with passwords. See below for format.Yes

note

The cdm config object is now deprecated. Instead, use the customerDetails object with a type of cdm.

If any paths in the configuration are given as relative paths, they will be relative to the current working directory.

Run​

Command Line Arguments​
FlagValid ValuesRequiredDefault
-confPath to the configuration fileYes

To run the server, execute the following command at a terminal:

java -jar <path to ednar jar> -c <path to config file>
