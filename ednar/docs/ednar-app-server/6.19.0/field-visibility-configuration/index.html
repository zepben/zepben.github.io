Field Visibility Configuration

This page provides further details on the field visibility configuration available for displaying/hiding fields in the UI. Click on any of the images below to view them full size in a new tab.

Setting and Adjusting Field Visibility Configuration​

Field visibility configuration is located in the field_configuration database table. If you have a database GUI tool, you may be able to edit the table directly, otherwise you will need to use SQL statements.

To set the initial value for field visibility, you will need to insert field name and visibility value into the table, as well as an option job type ID. The following SQL can be used:

Oracle:
INSERT INTO field_configuration (field_name, visible)
SELECT '<field name>', '<value>' FROM dual;

INSERT INTO field_configuration (field_name, visible, job_type_id)
SELECT '<field name>', '<value>', (SELECT id FROM job_types WHERE name = '<job type>') FROM dual;

MS SQL Server/PostgreSQL:
INSERT INTO field_configuration (field_name, visible)
SELECT '<field name>', '<value>';

INSERT INTO field_configuration (field_name, visible, job_type_id)
SELECT '<field name>', '<value>', (SELECT id FROM job_types WHERE name = '<job type>');
info

Field visibility does not affect any existing form validation - hiding a mandatory field may lead to invalid/unfilled NARs with no way to populate the required data.
