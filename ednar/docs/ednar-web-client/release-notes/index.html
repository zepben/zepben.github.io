[6.9.0]​

Breaking Changes​

  • None.

New Features​

  • Network variant layers (e.g. proposed, out of service) are now supported in the EDNAR map.
  • Customer list NMIs can now be downloaded as a CSV file.
  • New CUSTOMER_LIST_DEFAULT_COLUMNS app option added for configuring the default columns displayed in the customer list if the user has not set their own default.

Enhancements​

  • New OPEN_MAP_BY_DEFAULT app option for configuring whether the map opens by default when a NAR is displayed.

Fixes​

  • Fixed a device tree issue that caused incomplete customer information to be returned.
  • Upgraded network map version to fix several device tree issues:
    • Actions dropdown is now a small icon.
    • Customer nodes are now also highlighted.
    • Jump to Top/Bottom button is no longer animated.
  • Fixed an issue where removing apparatus to be covered did not remove the trace from the map.
  • Fixed issue where customer list supply point highlight layers on the map were not removed when the notification task was deleted.

Notes​

  • None.

[6.8.0]​

Breaking Changes​

  • All trace colours must now be defined in the traceColourConfig object in order to override the default trace colours.
  • ASSET_STRING_FORMAT option no longer supports equipmentContainers as a variable.

New Features​

  • Added support for admin users to add, edit, and delete custom keys.
  • Now uses network map search 2 component with the ability to filter cim classes
  • Added support for configuring all available reasons for each job type.
  • Added support for the following tracing capabilities:
    • Trace to upstream Switch
    • Trace to upstream Recloser
    • Downstream trace
  • Integrated device tree with access authority support for dynamic dropdown, node colors, and redirectTo option.
  • Updated Ednar to use latest Network Map changes with symbol rendering via getMapSymbolDetails and GraphQL client support.
  • Added support for custom messages in customer notification tasks.
  • Added ability to delete customers via polygon selection. Introduced GeoJSON include/exclude type for selection behavior while saving.
  • Added support to display NAR IDs in the customer list table for NMIs that exist in other NARs occurring at the same time.
  • Added a permission based access control for viewing customer sensitive information in the customer list table in the UI.
  • Added the functionality to customize customer list table columns:
    • Users can choose which columns to show.
    • Users can reorder columns as they like.
  • Added customer display via supply points with green-circle, polygon selection customer generation, visibility layer, and customer info modal.
  • Added support for hiding fields in the UI via field visibility configuration.
  • Added new Control Desk and Planning Area fields in the NAR Summary.
  • Added support to display the network summary from the isolation trace, configurable by authority type.
  • Added new callouts to NAR annotations and Access Authority annotations, which are saved separately to the NAR and allow better user control over positioning.
  • Callouts now have a label which allows them to be identified by a letter or series of letters.
  • Added the ability to create a NAR annotation callouts when adding an LV Switching Step from the map.
  • Added parent-child relationship for customer notification tasks.
  • Added symbols in device tree
  • Added the summary of the customer based on the total customers.
  • (LTS 6.6.X) Selected Depot and Region filters on the Dashboard are now also used to filter calendar events.
  • (LTS 6.6.X) New QUICK_SEARCH_OPENS_IN_NEW_TAB app option added. When set to true, NARs selected via quick search will open in a new browser tab.

Enhancements​

  • Can now specify which equipment containers to show in ASSET_STRING_FORMAT as well as Energizing Feeders and Energizing Substation.
  • Consolidated all trace colour configurations into a single traceColourConfig object.
  • Added support for tracking removed NMIs.
  • LV Switching Steon ps can now hold an optional label which can be used to refer to a callout the map, or can be used for any other purpose.
  • Added new Parent Task ID and Completion Comments search columns.
  • Updated styling, translation and layout for notification task headers.
  • Device tree now be resized.
  • LV assets will now have the energizing HV feeder used for the primary equipment container and clash detection.
  • Voltages below 1kV will now display as "LV" for asset text.
  • Ad-hoc traces can now be run from view mode.

Fixes​

  • Fixed an issue where isolation trace is highlighting beyond a downstream isolation trace when using multiple mid-span isolation points.
  • Fixed an issue where column colouring for Y/N answer was not functioning when pressing Apply button in the NAR Search column configuration modal.
  • Fixed an angle bracket decoding issues for AppOptions and generator rating.
  • Fixed an issue where start date and end date criteria filtered out when switching between table search view and calendar search view.
  • Fixed an issue where the header cut off some of the NAR form.
  • (LTS 6.6.X) NAR date ranger picker no longer requires "OK" button to be pushed between start and end dates. This also allows end dates to be changed even if start date is in the past.
  • (LTS 6.6.X) Search column config now resets to user/system default when toggling between Home and Search pages.
  • (LTS 6.6.X) Fixed system notification count badge being cutoff on the home page.
  • (LTS 6.6.X) Annotation colour picker now displays colour names correctly on hover.
  • (LTS 6.6.X) Selecting assets without equipment containers (i.e. poles) no longer displays errors if the allowed network level is set to NO_LV.
  • (LTS 6.6.X) 12 months of calendar events are now fetched for the NAR date picker instead of just 1.
  • (LTS 6.6.X) Fixed issue where "Is HTML" checkbox was not displaying or updating correctly in Admin Notification Template page.
  • (LTS 6.6.X) Fixed issue where Quick Search would open NARs in 2 new tabs.
  • (LTS 6.6.X) Fixed issue where ad-hoc Schedule print button didn't appear unless there was a matching NAR for the DMS ID in Quick Search.

Notes​

  • Replaced all hardcoded validation message with the appropriate translation keys.

[6.7.0]​

Breaking Changes​

  • None.

New Features​

  • Added support for auto-created customer notification tasks and add option to run them as bulk NAR operations from admin panel.
  • Added the functionality to select Access Authority Isolation Points to the following tabs:
    • HV Generators
    • LV Generators
    • LV Switching
  • Made customer notification delivery methods configurable per access type with threshold-based enable/disable logic.

    • Customer List isolation point selector
    • LV Generators isolation point selector
    • HV Generators isolation point selector

  • Added admin functionality to create, edit, view and delete webhooks. Requires ADMIN_WEBHOOK_CONFIG_READ to view and ADMIN_WEBHOOK_CONFIG_WRITE to edit/delete.
  • Added splitter functionality to split the NAR form and the asset map.

Enhancements​

  • A warning will appear if the selected isolation points on a customer list are outside any Access Authority.
  • A warning will appear when removing isolation points from Access Authorities, if assets exists in generators or customer notification tasks.
  • View recorded EIC details in notification checklist.
  • Removed all the PON references from the codebase.
  • Adds warning if customer list has multiple distributors.
  • An Alert will appear when duplicate NMIs are detected in another notification task.

Fixes​

  • The NAR form page will no longer periodically refresh in certain states when local storage is updated from another browser tab.
