Application

Once the New Request section has been chosen from the Home Page a pop-up will appear with the available NAR access types. These access types each have different lifecycles and business purposes. See NAR Lifecycles in the Admin documentation for more details.

Once you select the access type, you will see one of the below NAR creation modals. You can include as much or as little information now as you like, and fill the rest in throughout the application, however it's recommended to at least select an Access Reason and a Switching Type.

The new planned request wizard breaks down the information needed to create a network access request into various tabs. This allows information to be populated in a logical sequence. The wizard also auto populates data, where possible, thereby reducing data entry.

1. The information displayed in this section is described below:

A NAR open in view mode is shown with the eye icon.

2. The section contains the following fields:

  • Primary Zone - this is populated automatically from the Apparatus to be Covered field.
  • Location - this can either be populated from the selected NAR assets or via free text entry
  • Work Type - this is automatically populated using the Access Reason where applicable, however it can be changed either via the dropdown selector or free text entry
  • Access Type - once a NAR is created this cannot be changed
  • Access Reason - This must be selected from the dropdown list. This will also be used as the outage reason for customer notifications where applicable
  • Switching Type - This will determine the mandatory questions, lead times, attachments and participants. It will also change the default authorities.
  • Dates - You can select the start date and time and the end date and time for the NAR here. You can also choose it on the Calendar tab.

3. This section allows the end user to add participants. When adding a participant, the end user must select the participant type and name. Certain participants may be mandatory based on the selection of Switching Type.

An incomplete icon will be displayed on the participant header, if participant selection is incomplete.

4. This section is read only and shows the summary of information entered in the authorities tab.

5. This section allows the user to select if there are customer notifications required for the outages.

If there is a customer interruption, then the customer list tab is made available and a summary of the customer notification tasks in the customer list tab is shown in this section.

If there is no customer notification required, this section displays this.

6. This section shows all the mandatory questions that the user must answer.

In addition to this, it allows the user to add any optional questions to their application. The user can add optional questions by clicking on the Add Questions button.

7. This section allows for interested parties to be add to the NAR

The interested party must have a Name, Email Address, and Phone Number. Additional interested parties can be added by clicking the Add Interested Party button. Each interested party can be deleted or disabled.

8. The Attachments section has a field that allows for an attachment type to be selected then a document can be selected and uploaded to the NAR. Hovering over the icon will let the user know which attachments are mandatory. Certain questions may trigger further additional attachments. This section will also display attachments from other sections of the NAR, e.g. Authorities or Generators.
